## 錄音 12 新增重點

- 補強 `TCP / UDP` 的差異：TCP 像先確認再通話，UDP 像直接把訊息丟出去。
- 補強 `三向交握`：`SYN → SYN + ACK → ACK`，用來建立可靠連線並同步初始序號。
- 補強 `四次揮手`：`FIN → ACK → FIN → ACK`，因為 TCP 是雙向資料流，一方結束不代表另一方也立刻沒資料要送。
- 補強 `Sliding window` 與 `flow control`：接收端用 window 告訴傳送端「我還能收多少」，避免資料塞爆緩衝區。
- 補強 `TCP header` 欄位：`Sequence number`、`Acknowledgement number`、`Header Length`、`Flags`、`Window Size`、`Checksum`、`Urgent Pointer`、`Options`。
- 補充封包觀察方式：用類似 Wireshark 的封包欄位看 sequence、ACK、window 與資料長度如何互相對應。

## 錄音 13 新增重點

- 補強三向交握的封包觀察：`SYN` 會消耗一個 sequence number，所以對方回覆時 ACK 會是初始序號加 1。
- 補強四次揮手的封包觀察：`FIN` 也會消耗一個 sequence number，對方 ACK 時也會把 FIN 算進去。
- 補強 TCP header 的 flags：可以從 `SYN / ACK / FIN` 判斷封包是在建立、確認還是結束連線。
- 新增 [[TCP 壅塞控制]]：真正可傳送量不只看接收端 window，也要看 congestion window。
- 新增 `Slow Start`、`ssthresh`、`Congestion Avoidance`、`Duplicate ACK`、`Fast Retransmit`、`Fast Recovery`。
