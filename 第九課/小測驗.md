# 小測驗

## 題目

1. `TCP` 和 `UDP` 屬於 OSI 七層中的哪一層？
2. `TCP` 為什麼比 `UDP` 可靠？請至少寫出兩個原因。
3. `UDP` 為什麼常用在語音、影音或遊戲？
4. `IP address` 和 `Port number` 分別在解決什麼問題？
5. Port number 是幾 bits？範圍是多少？
6. `80`、`443`、`53` 分別常見於什麼服務？
7. UDP header 固定幾 bytes？有哪四個主要欄位？
8. TCP header 最少幾 bytes？為什麼比 UDP header 大？
9. `Sequence Number` 在 TCP 中代表什麼？
10. `ACK Number` 可以怎麼白話理解？
11. 如果 sequence number 是 `60000`，資料長度是 `1000 bytes`，接收端收到後 ACK 應該是多少？
12. 三向交握的三個步驟是什麼？
13. 三向交握為什麼不能簡單省成兩次？
14. 四次揮手的四個步驟是什麼？
15. 為什麼 TCP 結束連線通常需要四次揮手？
16. `Window Size` 欄位的用途是什麼？
17. `Flow control` 和 `Congestion control` 的差別是什麼？
18. `MSS` 和 `Window Scale` 大概在 TCP 中扮演什麼角色？
19. 為什麼 `SYN` 或 `FIN` 後面的 ACK number 通常會加 1？
20. `rwnd` 和 `cwnd` 分別代表什麼？真正可傳送量大致看哪一個？
21. `Slow Start` 和 `Congestion Avoidance` 的差別是什麼？
22. 什麼情況下 TCP 可能進行 `Fast Retransmit`？

## 答案

1. **傳輸層**。
2. **TCP 會建立連線、用 ACK 確認、用 sequence number 排序、遺失時重傳、用 window 控制流量**。
3. **因為 UDP header 小、沒有交握與確認流程，延遲低；即時資料通常可接受少量遺失**。
4. **IP 找主機；port 找主機上的應用程式或服務**。
5. **16 bits，範圍 0 ~ 65535**。
6. **80 = HTTP；443 = HTTPS；53 = DNS**。
7. **8 bytes；Source Port、Destination Port、Length、Checksum**。
8. **最少 20 bytes；因為 TCP 要放序號、確認號、控制旗標、window、checksum、options 等可靠傳輸資訊**。
9. **這段 TCP 資料在 byte stream 中的起始位置**。
10. **下一個期望收到的 byte 編號**。
11. **61000**。
12. **SYN → SYN + ACK → ACK**。
13. **避免舊的延遲請求被誤認成新連線，也讓雙方都確認收送方向與初始序號**。
14. **FIN → ACK → FIN → ACK**。
15. **TCP 是雙向資料流，一方送完不代表另一方也送完，所以兩個方向要分開關閉**。
16. **告訴對方目前還能接收多少資料，用來做 flow control**。
17. **Flow control 主要看接收端收不收得下；congestion control 主要看中間網路會不會太塞**。
18. **MSS 限制單一 TCP segment 可承載的資料量；Window Scale 用來放大 window size 的表示範圍**。
19. **因為 SYN 和 FIN 雖然通常不帶一般資料，但在 TCP 序號空間中會消耗 1 個 sequence number**。
20. **rwnd 是接收端宣告的可接收量；cwnd 是傳送端依網路壅塞狀態估計的可送量；真正可送量大致取 min(rwnd, cwnd)**。
21. **Slow Start 通常是初期快速、近似指數增加；Congestion Avoidance 是到門檻後比較保守、近似線性增加**。
22. **當傳送端收到多個重複 ACK，推測某段資料遺失時，可以不等 timeout 就快速重傳缺少的 segment**。
