# 快速記憶卡

## TCP / UDP

- `TCP`：可靠、連線導向、會確認、會重傳、會排序。
- `UDP`：快速、無連線、不保證送達、不保證順序。
- `TCP` 適合：網頁、檔案下載、Email、SSH。
- `UDP` 適合：DNS、語音、影音、遊戲、即時資料。

## Port

- `IP address`：找到哪一台主機。
- `Port number`：找到主機上的哪一支程式。
- Port 是 `16 bits`，範圍 `0 ~ 65535`。
- `0 ~ 1023`：Well-known ports。
- `49152 ~ 65535`：Dynamic / Private ports。
- `80`：HTTP。
- `443`：HTTPS。
- `53`：DNS。
- `21`：FTP。
- `25`：SMTP。

## UDP Header

- UDP header 固定 `8 bytes`。
- 四個欄位：Source Port、Destination Port、Length、Checksum。
- UDP 有 checksum，但不負責重傳。

## TCP Header

- TCP header 最少 `20 bytes`。
- `Sequence Number`：這段資料從哪個 byte 開始。
- `ACK Number`：下一個期望收到的 byte。
- `Header Length / Data Offset`：TCP header 長度，單位常用 4 bytes。
- `Window Size`：接收端目前還能收多少資料。
- `Checksum`：檢查 TCP segment 是否可能出錯。
- `Options`：可放 MSS、Window Scale 等資訊。

## Flags

- `SYN`：建立連線，同步初始序號。
- `ACK`：確認收到。
- `FIN`：結束某一方向的資料傳送。
- `RST`：重設連線。
- `PSH`：盡快推給應用程式。
- `URG`：有緊急資料。

## 連線流程

- 三向交握：`SYN → SYN + ACK → ACK`。
- 四次揮手：`FIN → ACK → FIN → ACK`。
- TCP 是 full-duplex，所以結束時兩個方向要分開處理。

## Sliding Window

- Stop-and-wait：傳一段等一次，效率低。
- Sliding window：一次允許多段資料在路上。
- ACK 回來後，window 往前滑。
- Flow control 看接收端是否收得下。
- Congestion control 看中間網路是否太塞。

## TCP 壅塞控制

- `rwnd`：接收端宣告還能收多少。
- `cwnd`：傳送端估計網路目前能承受多少。
- 可傳送量大致看 `min(rwnd, cwnd)`。
- `Slow Start`：一開始小心送，ACK 正常就快速增加 cwnd。
- `ssthresh`：slow start 和 congestion avoidance 的分界。
- `Congestion Avoidance`：接近門檻後改成比較保守的線性增加。
- `Duplicate ACK`：重複要求同一段，通常代表中間缺一段。
- `Fast Retransmit`：收到多個重複 ACK 後，不等 timeout 就重傳。
- `Fast Recovery`：丟包後降速，但不完全從零開始。

## 封包觀察

- `SYN` 會消耗 1 個 sequence number。
- `FIN` 也會消耗 1 個 sequence number。
- 所以看到 `SYN` 或 `FIN` 後，對方 ACK 通常會加 1。
