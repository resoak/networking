## 第1章 網路初探 / 基礎概念

### 填充題
1. `TCP/IP`
2. `星狀、直線型（匯流排）、環型、樹狀、混合式`
3. `PAN、LAN、MAN、WAN`

### 選擇題
1. `D`：Internet。
2. `C`：由全世界網路互連而成的是 Internet。
3. `D`：鍵盤不是網路範疇中的獨立網路設備。
4. `C`：Internet 早期基本功能之一是遠端登入（Telnet）。
5. `A`：依題意應為個人網路（PAN）。
6. `A`：辦公室網路通常屬於區域網路（LAN）。
7. `C`：依題意應為廣域網路（WAN）。
8. `B`：城市行政網路較接近都會網路（MAN）。
9. `A`：Bluetooth 屬於個人網路（PAN）。
10. `B`：Wi-Fi 屬於無線區域網路（WLAN / LAN）。
11. `D`：校園網路通常不是廣域網路。
12. `C`：若依一般教材標準答案，較常以星狀網路作答；原圖手寫為 `D`，老師若以「單一電腦故障不一定造成全網癱瘓」判斷，也可能接受。
13. `A`：FDDI 採環狀架構。
14. `D`：星狀網路若中心節點故障，整體網路會受影響。
15. `B`：直線型（Bus）需要終端器（Terminator）。
16. `C`：樹狀網路強調階層式觀念。
17. `C`：無線區域網路主要採 IEEE 802.11。
18. `A`：802.3 是有線乙太網路，不屬於無線媒體標準。
19. `B`：Internet 以 TCP/IP 為核心協定。
20. `C`：常作為參考依循架構的是 OSI。

### 問答題
#### 1. 名詞英文全名
- TCP/IP = **Transmission Control Protocol / Internet Protocol**
- bps = **bits per second**
- FTP = **File Transfer Protocol**
- WWW = **World Wide Web**
- VoD = **Video on Demand**
- PAN = **Personal Area Network**
- LAN = **Local Area Network**
- MAN = **Metropolitan Area Network**
- WAN = **Wide Area Network**
- DSL = **Digital Subscriber Line**
- LTE = **Long Term Evolution**
- ISO = **International Organization for Standardization**
- OSI = **Open Systems Interconnection**

#### 2. 何謂網路？
- 網路是把多台電腦、行動裝置、伺服器與周邊設備透過傳輸媒介連接起來的系統。
- 連上網路後，各節點可以交換資料、共享資源、提供服務。
- 簡要定義：**網路就是讓多個裝置彼此通訊與分享資源的連線架構。**

#### 3. 何謂 Internet？
- Internet 是由全球大量網路互相連接而成的「網路的網路」。
- 其核心通訊架構以 TCP/IP 協定族為基礎。
- 它提供 WWW、E-mail、FTP、遠端登入等服務。

#### 4. 依距離遠近分類並舉例
- **PAN**：手機連藍牙耳機、智慧手錶同步。
- **LAN**：家用 Wi-Fi、辦公室內部網路。
- **MAN**：市政府專網、跨校區連線。
- **WAN**：企業跨國專線、Internet。

#### 5. 何謂拓樸？常見類型
- 拓樸是指網路節點與傳輸線路的連接方式。
- 常見類型：星狀、直線型（匯流排）、環型、樹狀、混合式。

#### 6. 常見區域網路型態與速率
- Ethernet：10Mbps、100Mbps、1Gbps、10Gbps 以上皆常見。
- FDDI：100Mbps。
- Token Ring：舊式令牌環網。
- WLAN（802.11）：依 b/g/n/ac/ax 等規格不同而有不同速率。

#### 7. 常見廣域網路架構與速率
- 租用專線：穩定、延遲低。
- xDSL：利用電話線傳輸，上下行通常不對稱。
- Cable / FTTH：家庭寬頻常見。
- 4G / 5G：部署快，但受環境影響較大。

#### 8. 主從架構與對等架構差異
- **主從式（Client/Server）**：集中提供服務，易管理、好控管。
- **對等式（Peer-to-Peer）**：節點可直接共享資源，成本低但管理較分散。

#### 9. 何謂 OSI 模式？誰制定？有何作用？
- OSI 是 **ISO** 制定的七層參考模式。
- 作用是把複雜的網路通訊切成分層功能，便於設計、標準化與除錯。
- 七層依序為：實體層、資料連結層、網路層、傳輸層、會議層、呈現層、應用層。

## 第3章 傳輸媒介與配備

### 填充題
1. `降低電磁干擾與串音`
2. `傳送訊號`
3. `單模光纖、階梯型多模光纖、漸變型多模光纖`
4. `地波、天波、直達波`
5. `接頭、天線`
6. `ST 接頭、SC 接頭`
7. `數據機（Modem）`
8. `中繼器（Repeater）`
9. `集線器（Hub）、交換式集線器（Switch）`
10. `路由器（Router）`

### 選擇題
1. `C`：網路卡不是傳輸媒體。
2. `A`：互絞可降低電磁干擾與串音。
3. `C`：線長會影響使用效果，但不決定 CAT 等級本身。
4. `D`：同軸電纜中心導體負責傳送訊號。
5. `A`：同軸電纜的遮蔽層常為網狀導線。
6. `C`：光纖沒有金屬網狀導體。
7. `B`：光纖保密性最佳、不易受干擾。
8. `D`：光纖架設不算容易，設備與施工成本較高。
9. `D`
10. `A`：中繼器可延長傳輸距離、擴充網段。
11. `A`：多埠中繼器稱為集線器（Hub）。
12. `A`：多埠橋接器就是交換式集線器。
13. `B`：交換式集線器可減少不必要轉送。
14. `A`：交換式集線器本質上進行交換工作。
15. `D`：第三層交換器可執行封包路徑判斷。
16. `D`：中大型網路系統間的互連常用路由器。
### 問答題
#### 1. 名詞英文全名
- EMI = **Electromagnetic Interference**
- UTP = **Unshielded Twisted Pair**
- STP = **Shielded Twisted Pair**
- FTTH = **Fiber To The Home**
- LOS = **Line Of Sight**
- Modem = **Modulator / Demodulator**
- ADSL = **Asymmetric Digital Subscriber Line**
- NIC = **Network Interface Card**
- SNMP = **Simple Network Management Protocol**
- QoS = **Quality of Service**

#### 2. 雙絞線為何要互絞？
- 降低外部電磁干擾（EMI）。
- 降低線對之間的串音（Crosstalk）。
- 提升訊號穩定度與傳輸品質。

#### 3. UTP 與 STP 差異
- **UTP**：無遮蔽層，便宜、易施工，最常見。
- **STP**：有遮蔽層，抗干擾能力較強，但成本較高、接地要求較嚴。

#### 4. UTP 分類與特性
- Cat3：早期低速用途。
- Cat5 / Cat5e：100Mbps～1Gbps 常見。
- Cat6 / Cat6a：1Gbps～10Gbps，抗干擾較佳。

#### 5. 網路電纜各部分用途與原理
- 雙絞線靠互絞降低雜訊。
- 同軸電纜以中心導體傳訊、外層遮蔽抗干擾。
- 光纖靠全反射傳輸光訊號。

#### 6. 光纖優缺點與三種型態
- 優點：頻寬高、傳輸遠、抗干擾、保密性佳。
- 缺點：成本較高、施工與接續技術要求高。
- 型態：單模、階梯型多模、漸變型多模。

#### 7. 地波、天波、直達波
- 地波：沿地表傳播，常見於 AM 廣播。
- 天波：靠電離層反射，可做長距離通信。
- 直達波：需 LOS，常見於微波、衛星、Wi-Fi。

#### 8. RJ-11 與 RJ-45 差異
- RJ-11 常用於電話線，芯數較少。
- RJ-45 常用於乙太網路，8P8C，支援資料傳輸。

#### 9. 【實作/市調題】不同纜線、接頭及價格分析範例
- UTP Cat5e：價格低，適合一般 1Gbps 環境。
- UTP Cat6：價格中等，抗干擾較佳。
- 光纖：價格較高，但適合長距離與高頻寬骨幹。

#### 10. 為何需要數據機？
- 負責數位與類比訊號之間的轉換。
- 讓原本不直接相容的傳輸媒介能承載電腦資料。

#### 11. 為何需要中繼器？
- 長距離傳輸會造成訊號衰減與失真。
- 中繼器會重新整形、放大並再生訊號。

#### 12. 為何需要集線器？與中繼器差異？
- 集線器可看成**多埠中繼器**。
- 它把訊號廣播到所有埠，供多台設備共用。

#### 13. 一般型集線器與交換式集線器差異
- 一般型集線器：所有埠共享頻寬，容易碰撞。
- 交換式集線器：依 MAC 位址轉送，碰撞少、效率高。
- 智慧型集線器：另含管理與監控功能。

#### 14. 橋接器用途與原理
- 用來連接兩個網段。
- 依 MAC 位址學習節點位置，進行過濾與轉送。

#### 15. 交換式集線器原理與差異
- 根據 MAC 位址表決定封包要送到哪個埠。
- 相較 Hub，效率更高且每埠可形成獨立碰撞網域。

#### 16. 【實作/市調題】網路卡與交換器價格比較範例
- 1GbE 網卡：價格低、適合一般桌機。
- 2.5GbE / 10GbE 網卡：適合高速 NAS 或伺服器。
- 可比較項目：埠數、速率、是否支援 VLAN / QoS / PoE、價格。

#### 17. 路由器用途與原理
- 路由器負責不同網路之間的封包轉送。
- 它根據路由表決定下一跳路徑。

#### 18. 圖 3.22 與圖 3.25 差異
- 圖 3.22 是「以路由器將幾個網路系統連接起來」的概念示意圖，著重路由器把甲、乙、丙三個獨立網路互連。圖 3.25 則是實際的廣域網路連線架構，著重各地分公司先接到路由器，再透過專線數據機與廣域網路相連。簡單說，圖 3.22 偏概念拓樸圖；圖 3.25 偏實際 WAN／租用專線應用圖。

#### 19. 【實作/市調題】路由器規格與價格差異
- 低階家用型：一般上網。
- 中階型：支援雙頻/三頻、MU-MIMO、Mesh。
- 高階型：支援 Wi-Fi 6/6E、2.5GbE、VLAN、VPN、進階 QoS。

#### 20. Router、Route、Routing 差異
- **Router**：路由器，實體設備。
- **Route**：路由，指封包前進的路徑。
- **Routing**：路由處理/尋徑過程。

#### 21. 第三層交換器與路由器差異
- L3 Switch 可在交換器硬體基礎上做 IP 封包轉送。
- 路由器功能通常更完整，支援更多 WAN、NAT、ACL、VPN。

#### 22. 依目前網路系統說明設備與架構
- 家用範例：`ISP -> 光纖數據機 -> 路由器/AP -> Switch -> PC / NAS / 手機`。
- 校園/公司範例：`ISP 專線 -> 防火牆 -> 核心交換器 -> 樓層交換器 -> 終端設備`。

## 第4章 通訊協定 / OSI

### 填充題
1. `協定（Protocol）`
2. 完整七層為：`實體層、資料連結層、網路層、傳輸層、會議層、呈現層、應用層`
3. `片段（Segment）` 或一般稱 `封包片段`
4. `LLC、MAC`
5. `IETF、IRTF`
6. `提案標準（Proposed Standard）、草案標準（Draft Standard）、正式標準（Standard）`

### 選擇題
1. `B`：OSI 七層是參考模式。
2. `D`：公司、國際組織、政府機關都可以參與標準制定。
3. `D`：加解密通常歸在呈現層。
4. `C`：訊框（Frame）對應資料連結層。
5. `B`：Segment 是傳輸層概念。
6. `D`：距離/成本量測可綜合頻寬、延遲、可靠度等。
7. `C`：FTP 屬應用層，不是傳輸層。
8. `C`：會議層負責建立、維持、終止會話。
9. `A`：Internet 採 TCP/IP。
10. `D`：IEEE 將資料連結層切成 LLC 與 MAC。
11. `A`：資料編碼、壓縮、加密屬呈現層。
12. `B`：閘道器功能最強，可跨不同協定系統工作。

### 問答題
#### 1. 名詞英文全名
- IEEE = **Institute of Electrical and Electronics Engineers**
- ANSI = **American National Standards Institute**
- ITU = **International Telecommunication Union**
- CCITT = **Consultative Committee for International Telegraph and Telephone**
- EIA/TIA = **Electronic Industries Alliance / Telecommunications Industry Association**
- ISO = **International Organization for Standardization**
- IAB = **Internet Architecture Board**
- IETF = **Internet Engineering Task Force**
- IRTF = **Internet Research Task Force**
- OSI = **Open Systems Interconnection**
- RS-232 = **Recommended Standard 232**
- TCP/IP = **Transmission Control Protocol / Internet Protocol**
- LLC = **Logical Link Control**
- MAC = **Media Access Control**
- RIP = **Routing Information Protocol**
- OSPF = **Open Shortest Path First**
- UDP = **User Datagram Protocol**
- RFC = **Request for Comments**

#### 2. 何謂 Protocol？
- Protocol 是通訊雙方共同遵守的規則，規範資料格式、順序、錯誤處理與同步方式。

#### 3. OSI 分哪幾層？是不是標準？
- 分為七層：實體、資料連結、網路、傳輸、會議、呈現、應用。
- 它不是單一「通訊協定」，而是**參考模式**。

#### 4. OSI 各層重點整理
- 應用層：HTTP、FTP、SMTP。
- 呈現層：編碼、加密、壓縮。
- 會議層：建立/管理會話。
- 傳輸層：TCP、UDP。
- 網路層：IP、Router。
- 資料連結層：Ethernet、Switch。
- 實體層：Cable、Hub、Repeater。
#### 5. 資料封包組成處理機制
- 發送端由上往下逐層加上控制資訊，稱為**封裝**。
- 接收端由下往上拆解，稱為**解封裝**。

#### 6. 資料連結層功能與 IEEE 子層
- 負責成框、MAC 定址、錯誤偵測與媒體存取控制。
- IEEE 將其分為 LLC 與 MAC。

#### 7. 路由中「距離」量測依據
- 常見依據有：跳數、延遲、頻寬、成本、可靠度、負載。

#### 8. RIP 與 OSPF 差異
- RIP：距離向量、跳數為主、簡單但收斂較慢。
- OSPF：連線狀態、依成本計算最短路徑、收斂快、適合大型網路。

#### 9. 圖 4.10 最佳路徑
- 節點 **E 到 C** 的理想路徑是 **E → G → F → C**。
- 原因是這條路徑的距離總和最小：`E→G = 1`、`G→F = 2`、`F→C = 2`，合計 **5**。
- 其他常見路徑如 `E → D → A → C` 的總和為 `3 + 2 + 1 = 6`，比 5 大，所以不是最佳路徑。

#### 10. 傳輸層功能與可靠傳輸
- 功能：分段、重組、流量控制、錯誤控制、連線管理。
- TCP 可靠傳輸靠序號、ACK、重傳、視窗控制完成。

#### 11. 呈現層功能與例子
- 編碼轉換：ASCII、Unicode。
- 加密解密：TLS/SSL。
- 壓縮解壓：JPEG、MPEG、ZIP。

#### 12. 各設備與 OSI 的關係
- 中繼器：實體層。
- 交換式集線器：資料連結層。
- 路由器：網路層。
- 閘道器：可跨多層甚至應用層。

#### 13. TCP/IP 與 OSI 關係
- TCP/IP 不只有 TCP 與 IP，還包含 HTTP、FTP、UDP、ARP、ICMP 等大量協定。
- 它是 Internet 最常用的協定族，可大致對應到 OSI 各層。

#### 14. 何謂 RFC？在哪裡找？
- RFC 是 Internet 技術文件集合。
- 可在 `rfc-editor.org` 或 IETF 官網查詢。

## 第5章 區域網路架設

### 填充題
1. `10Mbps、Broadband（寬頻）、3600`
2. `1000Mbps、Baseband（基頻）、Twisted Pair（雙絞線）`
3. `Full Duplex（全雙工）`
4. `藍、橘、綠、棕`
5. `白綠、綠、白橘、藍、白藍、橘、白棕、棕`
6. `0.5`
7. `100 公尺`
8. `GBIC 或 SFP 光纖模組插槽`
9. `WPAN、WLAN、WMAN`
10. `Bluetooth、ZigBee`
11. `WWAN`
12. `WEP、WPA（或 WPA2）`
13. `Ad Hoc、Infrastructure`
14. `SSID`
15. `橋接`
16. `漫遊（Roaming）`
17. `中繼（Repeater）`

### 選擇題
1. `D`：AT&T 未參與 DIX 乙太網路標準制定。
2. `D`：應為 10Base5，不是 100Base5。
3. `A`：1000BaseT 使用雙絞線。
4. `C`：UTP 不會有紅色這種標準線對色。
5. `B`：乙太網路接頭是 RJ-45。
6. `D`：護套可保護、固定，也常用於辨識。
7. `A`：1000BaseT 會用到 8 條線。
8. `B`：集線器與電腦節點間通常採直接線，不是跳線纜線。
9. `C`：彎曲半徑通常應大於 1 吋。
10. `A`：接頭末端未絞部分應小於 0.5 吋。
11. `D`：Wi-Fi 是無線區域網路。
12. `A`：IEEE 802.3 是有線乙太網路。
13. `C`：802.11g 使用 2.4GHz。
14. `C`：兩種模式主要差在有沒有 AP。

### 問答題
#### 1. 名詞英文全名
- DIX = **Digital / Intel / Xerox**
- UTP = **Unshielded Twisted Pair**
- EMI = **Electromagnetic Interference**
- MDI-X = **Medium Dependent Interface Crossover**
- GBIC = **Gigabit Interface Converter**
- SFP = **Small Form-factor Pluggable**
- WMAN = **Wireless Metropolitan Area Network**
- WLAN = **Wireless Local Area Network**
- WPAN = **Wireless Personal Area Network**
- ISM = **Industrial, Scientific and Medical**
- MIMO = **Multiple Input Multiple Output**
- Wi-Fi = **Wi-Fi（常見教材會寫 Wireless Fidelity，但這不是正式技術全名）**
- LoRa = **Long Range**
- WEP = **Wired Equivalent Privacy**
- WPA = **Wi-Fi Protected Access**
- AP = **Access Point**
- IBSS = **Independent Basic Service Set**

- BSS = **Basic Service Set**

#### 2. Ethernet 為何叫 Ethernet？
- 借用「乙太（Ether）」作為共享媒介的概念，表示訊號像在共同媒體中傳遞。

#### 3. 乙太網路發展沿革
- 起源於 Xerox，後由 DIX 推動，再由 IEEE 制定 802.3 標準，演進到 10M、100M、1G、10G 以上。

#### 4. 乙太網路普及原因
- 開放標準、成本低、設備多、架設與維護較容易。

#### 5. Xerox 放棄商標權的影響
- 有助於標準開放與產業普及，促進更多廠商加入。

#### 6. 各種乙太網路傳輸系統整理
- 10Base2：10Mbps、同軸、約 185m。
- 10Base5：10Mbps、同軸、約 500m。
- 10BaseT：10Mbps、雙絞線、100m。
- 100BaseTX：100Mbps、雙絞線、100m。
- 1000BaseT：1Gbps、雙絞線、100m。

#### 7. 【實作/市調題】材料與配備調查範例
- 可列：網路線、RJ-45 接頭、壓線鉗、測線器、交換器、網路卡、AP。
- 比較欄位：品牌、規格、單價、適用速率、優缺點。

#### 8. 若要自己架設網路會選哪些材料？
- Cat6 網路線、RJ-45 接頭、8 埠 Gigabit Switch、Wi-Fi 6 AP、壓線鉗、測線器。

#### 9. RJ-45 壓線前後差異
- 壓線前：線芯未固定、接觸不穩。
- 壓線後：針腳刺入線芯形成導通並固定外被。

#### 10. RJ-45 護套功能
- 保護卡榫、增加拉力緩衝、方便顏色分類與管理。

#### 11. 4 埠、總頻寬 800Mbps 代表什麼？
- 代表每埠可能是 100Mbps 全雙工。
- 4 埠 × 100Mbps × 2（全雙工）= 800Mbps。

#### 12. 【實作題】壓線實習範例
- 使用測線器檢查 1~8 芯是否全通。
- 若燈號順序正常，表示壓線成功。

#### 13. TIA/EIA 568 在不同連接情況的做法
- 網路卡對網路卡：交叉線（A 對 B）。
- 網路卡對集線器：直通線（A-A 或 B-B）。
- 集線器對集線器：傳統上交叉線；若支援 uplink / Auto MDI-X，可用直通線。
#### 14. 兩台集線器直接用直通線的條件
- 其中一台需具備 **uplink 埠** 或 **Auto MDI-X** 功能。

#### 15. Auto Sensing 與 Auto MDI-X
- **Auto Sensing**：自動偵測連線速率/雙工模式。
- **Auto MDI-X**：自動判斷並切換直通/交叉，不必再手動選線。

#### 16. 【觀察題】現場網路改善建議範例
- 線材標籤化、核心設備集中管理、交換器取代 Hub、AP 分頻與分區避免干擾。

#### 17. Giga 以上乙太網路架設方式
- 端點採 Cat5e/Cat6 以上線材，骨幹可使用光纖，交換器與網卡需支援對應速率。

#### 18. 無線網路依大小分類與用途
- **WPAN**：藍牙、穿戴裝置。
- **WLAN**：家用/校園/辦公 Wi-Fi。
- **WMAN / WWAN**：都會/廣域無線接取。

#### 19. 何謂 Wi-Fi？
- Wi-Fi 是以 IEEE 802.11 為核心的無線區域網路技術。
- 特性：無線接取、移動便利、部署快，但易受干擾與遮蔽物影響。

#### 20. Bluetooth 與 ZigBee 差異
- Bluetooth：短距離、常見於耳機/手機。
- ZigBee：低功耗、低速率、適合感測器與 IoT。

#### 21. WLAN 應用場合、限制與常見規格
- 應用：家中、學校、辦公室、公共熱點。
- 限制：易受干擾、距離有限、安全性需妥善設定。
- 常見規格：802.11b/g/n/ac/ax。

#### 22. Ad Hoc 與 Infrastructure 差異
- **Ad Hoc**：設備彼此直接連線，沒有 AP。
- **Infrastructure**：由 AP 統一管理接入。
- 實務上通常選 Infrastructure，管理與安全都較好。

#### 23. 何謂 ESS？如何漫遊？
- ESS 是多個使用同一 SSID 的 BSS 所組成的延伸服務區。
- 使用者在不同 AP 間移動時，可透過漫遊維持連線。

#### 24. AP / 中繼 / 橋接模式差異
- AP 模式：提供終端設備無線接入。
- 中繼模式：延伸原有無線訊號範圍。
- 橋接模式：把兩個網段或兩棟建築以無線方式橋接起來。

## 第7章 區域網路原理

### 填充題

11. `RTS、CTS`
12. `WEP`

### 選擇題
1. `A`：IEEE 802 主要規範 LAN。
2. `B`：對應 OSI 的實體層與資料連結層。
3. `C`：802.2 負責 LLC。
4. `D`：CSMA/CD 不是無線 LAN 機制，也不是 WAN 存取方式或 Internet 的主要機制，因此應選以上皆非。
5. `C`：MAC 位址是實體位址。
6. `A`：Interframe Gap 用來區隔前後訊框與傳送時序。
7. `B`：最大 1518 Bytes、最小 64 Bytes。
8. `A`：退讓時間不是固定值。
9. `B`：2500 × 10 / 80 = 312.5 公尺。
10. `C`：802.11 用的是 CSMA/CA，不是 CSMA/CD。
11. `B`：Beacon。
12. `D`：免競爭週期 + 競爭週期 = 超級訊框。
13. `D`：Infrastructure 可搭配 PCF、DCF、CSMA/CA。
14. `D`：EIFS 不是最短，SIFS 才最高優先且最短。
15. `C`：CW = 競爭視窗。
16. `D`：控制、管理、資料訊框都屬於 802.11 訊框類型。
17. `D`：To DS 與 From DS 都為 1 常見於 WDS / AP 間轉送情境。

### 問答題
#### 1. 名詞英文全名
- IEEE = **Institute of Electrical and Electronics Engineers**
- LAN = **Local Area Network**
- MAN = **Metropolitan Area Network**
- HLI = **Higher-Layer Interface**
- LLC = **Logical Link Control**
- MAC = **Media Access Control**
- CSMA/CD = **Carrier Sense Multiple Access with Collision Detection**
- CSMA/CA = **Carrier Sense Multiple Access with Collision Avoidance**
- DIX = **Digital / Intel / Xerox**
- SFD = **Start Frame Delimiter**
- OUI = **Organizationally Unique Identifier**
- ARP = **Address Resolution Protocol**
- RARP = **Reverse Address Resolution Protocol**
- DSAP = **Destination Service Access Point**
- SSAP = **Source Service Access Point**
- FCS = **Frame Check Sequence**
- CRC = **Cyclic Redundancy Check**
- PCF = **Point Coordination Function**
- DCF = **Distributed Coordination Function**
- IFS = **Interframe Space**
- RTS = **Request To Send**
- CTS = **Clear To Send**
- CW = **Contention Window**
- NAV = **Network Allocation Vector**

#### 2. IEEE 802 工作群組與 OSI 關係
- IEEE 802 主要對應 OSI 的實體層與資料連結層。
- 802.1：整體架構、橋接、管理；802.2：LLC；802.3：乙太網路；802.11：無線網路。

#### 3. IEEE 802.3 補充文件規範什麼？
- 規範不同乙太網路媒介、速率、編碼方式與實體層細節。

#### 4. DIX 2.0 與 IEEE 802.3 差異
- 主要差在資料鏈結層格式中的 **Type / Length** 欄位解讀。
- Ethernet II（DIX）在 IP 網路中仍很常見。

#### 5. 何謂 CSMA/CD？
- 先聽再傳，多節點共享媒介；若碰撞則偵測、送出壅塞訊號、隨機退讓後重送。

#### 6. 何謂壅塞訊號（Jamming Signal）？
- 碰撞發生後送出的特殊訊號，用來通知所有節點碰撞已發生。

#### 7. 實體位址與邏輯位址
- 實體位址：MAC。
- 邏輯位址：IP。

#### 8. MAC 位址規範方式
- 共 48 位元，前 24 位元通常是 OUI，後 24 位元由廠商分配。

#### 9. 實體廣播位址
- `FF:FF:FF:FF:FF:FF`，用於同一廣播網域內向所有節點廣播。

#### 10. 小於 64 Bytes 或大於 1518 Bytes 如何處理？
- 太小：補齊（Padding）到最小長度。
- 太大：在乙太網路中通常會被視為 oversized frame（giant）而丟棄；IP 層的分片是發生在封包送上實體媒介之前。

#### 11. IEEE 802.2 負責哪些工作？
- 負責 LLC 子層，提供上層統一介面並協調不同 MAC 技術。

#### 12. 【實作題】Wireshark 抓包解析
- 可擷取 ARP、ICMP、DNS、TCP 三向交握等封包，說明來源/目的 MAC、IP、Port、旗標與用途。
#### 13. 何謂碰撞網域與廣播網域？
- 碰撞網域：會互相產生碰撞的範圍。
- 廣播網域：廣播訊框可傳到的範圍。
- Switch 可切分碰撞網域；Router 可切分廣播網域。

#### 14. 何謂 Interframe Gap？
- 訊框與訊框間的最小間隔時間，用來讓設備恢復/同步。
- 在乙太網路中常以 **96 bit times** 表示。

#### 15. 何謂 Collision？如何判定？
- 多個節點同時傳送造成訊號重疊。
- 送端在傳送時若偵測到媒介訊號異常，即可判定碰撞。

#### 16. 為何最小訊框是 64 Bytes？
- 為了保證送端在最壞情況下仍來得及偵測碰撞。

#### 17. 15Mbps 新型乙太網路計算
- 7 段 × 200m，傳播延遲約 `7 μs`。
- 6 個 Repeater，每個 `2.15 μs`，共 `12.9 μs`。
- 再加整體上升時間 `5 μs`，單程約 `24.9 μs`。
- 來回延遲約 `49.8 μs`。
- **Time Slot ≈ 49.8 μs**。
- 最小訊框位元數約 `15Mbps × 49.8μs ≈ 747 bits`，可寫成 **約 750 bits / 約 94 Bytes**。

#### 18. 何謂 Time Slot？
- 是乙太網路碰撞偵測與退讓計算的重要時間單位。
- 退讓時間通常是 `隨機值 × Time Slot`。

#### 19. 10Mbps 可 2500m，50Mbps 可多遠？
- `2500 × 10 / 50 = 500 公尺`。

#### 20. 1Gbps 乙太網路最小封包大小
- 若談半雙工 Gigabit Ethernet 的碰撞偵測，常會提到 **512 Bytes 的延伸最小傳輸長度**。
- 但標準 MAC frame 基本最小值仍是 64 Bytes。

#### 21. 802.11 集中式與分散式架構
- **PCF（集中式）**：由 AP/協調者控制傳送次序。
- **DCF（分散式）**：由各站競爭媒介。
- CSMA/CA 屬於分散式（DCF）核心機制。

#### 22. 超級訊框、免競爭模式、競爭模式
- 超級訊框由免競爭期（CFP）與競爭期（CP）組成。
- 免競爭模式由 AP 輪詢；競爭模式由各節點用 CSMA/CA 競爭媒介。

#### 23. 802.11 的 IFS 種類
- SIFS：最短，優先權最高。
- PIFS：PCF 使用。
- DIFS：DCF 使用。
- EIFS：錯誤後使用，時間較長。

#### 24. 802.11 的槽格時間
- 用於退讓計時的基本單位，競爭視窗與倒數都依它計算。

#### 25. 802.11 的退讓時間
- `Backoff Time = Random(0 ~ CW) × Slot Time`

#### 26. 802.11 重傳機制
- 發送後若未收到 ACK，視為失敗；經退讓後再重送，超限後丟棄。

#### 27. CSMA/CA 如何確保傳送成功？
- 先聽媒介，空閒後等待 IFS，再執行隨機退讓；必要時加 RTS/CTS 與 ACK。

#### 28. 何謂隱藏節點？
- 兩個節點彼此聽不到對方，但都能聽到 AP，容易造成碰撞；可用 RTS/CTS 改善。

#### 29. 802.11 訊框控制欄位細項
- 包含：協定版本、Type、Subtype、To DS、From DS、More Frag、Retry、Power Management、More Data、Protected Frame、Order。

#### 30. 802.11 位址欄位如何使用？
- 最多可有 4 個位址欄位，視 To DS / From DS 決定來源、目的、發送端、接收端的對應意義。

#### 1. 名詞英文全名
- IP = **Internet Protocol**

- CIDR = **Classless Inter-Domain Routing**
- DHCP = **Dynamic Host Configuration Protocol**
- DNS = **Domain Name System**
- FQDN = **Fully Qualified Domain Name**
- TLD = **Top Level Domain**
- TWNIC = **Taiwan Network Information Center**

- DS = **Differentiated Services**
- DSCP = **Differentiated Services Code Point**
- QoS = **Quality of Service**

- MTU = **Maximum Transmission Unit**
- TTL = **Time To Live**
- ng = **next generation**
- IoT = **Internet of Things**
- IID = **Interface Identifier**

#### 2. 何謂 IP？IPv4 / IPv5 / IPv6 / IPng
- IP 是網際網路上的邏輯定址與封包遞送協定。
- IPv4：32 位元；IPv5：非主流實驗版本；IPv6 / IPng：128 位元，為解決位址不足而生。

#### 3. Octet 與 Byte 差異
- Octet 明確指 8 位元；Byte 在現代幾乎也視為 8 位元。

#### 4. IPv4 位址分類方式
- 可依 A/B/C/D/E 類分類，也可依公有/私有、單播/廣播/群播、CIDR 分類。

#### 5. 何謂網域位址？
- 用來識別某一網段本身；主機位元全為 0 時即為網路位址。

#### 6. 何謂廣播位址？
- 用來對某一網段內所有主機廣播；主機位元全為 1 即為廣播位址。

#### 7. 何謂 Multicasting？何謂 MRouter？
- Multicasting 是送給特定一群接收者；MRouter 是支援多播路由的路由器；IPv4 多播使用 Class D 位址。

#### 8. 何謂網路遮罩？
- 用來區分 IP 中哪些位元屬於網路、哪些屬於主機；與 IP 做 AND 可得網路位址。

#### 9. IPv4 重要保留位址
- `0.0.0.0`、`127.0.0.1`、`255.255.255.255`，以及私有位址區塊 10/8、172.16/12、192.168/16。

#### 10. 何謂 NAT？
- 把內部私有 IP 轉換成外部公有 IP，可節省公有位址並隔離內外部網路。

#### 11. 合法 IP 與類型判斷
- `160.32.88.250`：合法，B 類。
- `224.2.25.38`：合法，D 類（多播）。
- `0.0.0.0`：保留位址。
- `500.2.87.62`：不合法。
- `192.168.132.58`：合法，C 類私有位址。
- `210.32.96.170`：合法，C 類公有位址。
- `140.134.256.30`：不合法。
- `10.25.125.5`：合法，A 類私有位址。
- `127.0.0.1`：合法，Loopback。
- `921.921.921.921`：不合法。

#### 12. 子網路規劃方式與目的
- 從主機位元借位元切出更多子網。
- 目的：分散管理、降低廣播、提升安全與效率。

#### 13. 【依實際環境】子網路規劃作答範例
- 先統計部門/樓層/設備數，再為每個區段分配對應大小的子網，並預留成長空間。

#### 14. 何謂 CIDR？
- CIDR 是無類別網域間路由，以 `/前綴長度` 表示網路大小，提升位址使用效率。

#### 15. 何謂 Supernet？
- 把多個連續網段整併成較大的區塊，可減少路由表項目；與 Subnet 相反。

#### 16. CIDR 區塊選取規範
- 位址數量通常需為 2 的次方，起始位址必須對齊該區塊大小。

#### 18. 何謂 DHCP？
- DHCP 會自動發放 IP、遮罩、閘道器、DNS 等設定，適合大量終端設備環境。

#### 19. 何謂 DNS？
- DNS 把人可讀的網域名稱轉成 IP 位址。

#### 20. 網域名稱、根網域、頂層網域、FQDN
- Root Domain 是最頂端；TLD 如 `.com`、`.tw`；FQDN 如 `www.example.com.`。

#### 21. 何謂短網址？何謂泛用型網域名稱？
- 短網址是把長網址映射成較短網址；gTLD 如 `.com`、`.org`、`.net`。

#### 22. Domain 與 Zone 差異
- Domain 是命名空間概念；Zone 是 DNS 伺服器實際管理的一段資料範圍。

#### 23. DNS 查詢有哪些類型？
- 遞迴查詢：被問的一方要幫你問到底。
- 反覆/迭代查詢：被問的一方只告訴你下一個該問誰。

#### 24. 何謂動態 DNS？
- IP 變動時自動更新 DNS 對應，適合家用寬頻與遠端監控。

#### 25. IP 標頭欄位定義
- 依圖中的 Wireshark 畫面，此 IP 標頭可整理為：
  - **Version = 4**：表示此封包使用 IPv4。
  - **IHL / Header Length = 5**：代表 IP 標頭長度為 `5 × 32 bits = 20 bytes`。
  - **Service Type = Routine / Delay-Normal / Throughput-Normal / Reliability-Normal**：表示此封包未特別要求高優先權、低延遲或高可靠度，屬一般傳輸。
  - **Total Length = 54 octets**：整個 IP 封包總長度為 54 Bytes，包含 IP 標頭與資料。
  - **Fragment ID = 9276**：分片識別碼；若封包被切成多片，接收端可用此欄位判斷哪些分片屬於同一原始封包。
  - **Flags = Do not fragment, Last fragment**：表示此封包設定為不要分片，且目前也是最後一片；因為沒有實際分片，所以可視為單一完整封包。
  - **Fragment Offset = 0**：表示這個封包不是某個分片封包中的中間段或尾段，而是從開頭開始。
  - **TTL = 255**：封包最多可經過 255 次路由轉送，每經過一台路由器會遞減，降到 0 就會被丟棄。
  - **Protocol = TCP (0x06)**：表示 IP 資料負載上層使用的是 TCP 協定。
  - **Header Checksum = 0xF5D2**：用來檢查 IP 標頭在傳送過程中是否發生錯誤。
  - **Source IP = 140.134.36.56**：來源主機位址。
  - **Destination IP = 140.134.36.110**：目的主機位址。
  - **Options = None**：此封包沒有使用 IP 選擇項，因此標頭維持最基本的 20 Bytes。
- 若要補圖中的空格，可直接寫成：
  - `Version = 4`
  - `Header Length = 5 (20 bytes)`
  - `Service Type = Normal`
  - `Total Length = 54 bytes`
  - `Fragment ID = 9276`
  - `Flags = Do not fragment`
  - `Fragment Offset = 0`
  - `TTL = 255`
  - `Protocol = TCP (0x06)`
  - `Checksum = 0xF5D2`
  - `Source IP = 140.134.36.56`
  - `Destination IP = 140.134.36.110`
  - `Options = None`

#### 26. IP 封包優先權如何設定？
- IPv4 常透過 TOS / DSCP；IPv6 常透過 Traffic Class / DSCP 搭配其他機制處理，Flow Label 可用於識別特定 flow，但不宜視為唯一依據。

#### 27. IHL 與 TL 差異
- IHL：Header 長度，單位為 32 位元字。
- TL（Total Length）：整個 IP 封包長度，單位為位元組。

#### 28. 何謂 MTU？
- 單一鏈路可承載的最大封包大小；超過時需分片或調整。

#### 29. FO 與 ID 如何運作？
- FO 表示分片在原封包中的位置，單位是 8 Bytes；ID 則讓接收端知道哪些分片屬於同一原始封包。

#### 30. 1 補數演算法與標頭檢查碼
- 1 補數演算法的作法，是先把 **IP 標頭以 16 位元為一組** 依序相加；若加總結果超過 16 位元，就把最高位的進位加回最低位；全部加完後，再對結果取 **1 補數**，就是標頭檢查碼。
- 依第 25 題封包內容，可將 IP 標頭寫成下列 16 位元資料（檢查碼欄位先視為 `0000`）：
  - `4500`
  - `0036`
  - `243C`
  - `4000`
  - `FF06`
  - `0000`
  - `8C86`
  - `2438`
  - `8C86`
  - `246E`
- 逐步相加如下：
  - `4500 + 0036 = 4536`
  - `4536 + 243C = 6972`
  - `6972 + 4000 = A972`
  - `A972 + FF06 = 1A878 → A879`（超過 16 位元，將進位 1 加回）
  - `A879 + 0000 = A879`
  - `A879 + 8C86 = 134FF → 3500`
  - `3500 + 2438 = 5938`
  - `5938 + 8C86 = E5BE`
  - `E5BE + 246E = 10A2C → 0A2D`
- 最後對 `0A2D` 取 1 補數：
  - `FFFF - 0A2D = F5D2`
- 所以本題 **IP Header Checksum = `0xF5D2`**，與圖中 Wireshark 顯示結果一致。

#### 31. IP 選擇項 Code 欄位定義
- Code 通常由 `1 bit Copy Flag + 2 bits Class + 5 bits Number` 組成。

#### 32. Record Route 選項
- 功能：記錄封包經過的路由節點。

#### 33. Source Routing 選項
- 讓傳送端指定封包要經過的路徑，可分嚴格與鬆散兩種。

#### 34. Strict 與 Loose Source Routing 差異
- Strict：每一跳都被明確指定。
- Loose：只規定必經節點，中間可自行尋徑。
- `137 = 10001001₂`，`131 = 10000011₂`。

#### 35. Timestamp 選項目的與時間基準
- 用來記錄封包經過節點的時間資訊，常以毫秒表示，通常以 UTC / GMT 為基準。

#### 36. Timestamp 欄位與運作機制
- 常含 Type、Length、Pointer、Overflow/Flag、Timestamp Data；封包經過節點時依規則寫入時間資訊。

#### 37. IPv6 位址空間與表示方式
- 128 位元，位址空間極大；可分單播、多播、任點播；用 16 進位表示，可用 `::` 壓縮。

#### 38. 何謂 IoT？
- IoT 是把感測器、裝置、網路與雲端服務整合起來，例如智慧家電、智慧監控、智慧工廠。

#### 39. IPv6 如何透過 IPv4 網路通訊？
- 主要做法是 **Tunneling**：把 IPv6 封包封裝在 IPv4 封包中傳送。

#### 40. IPv4 相容位址與 6to4 位址
- IPv4 相容位址：早期遷移用途，現在多視為過時或極少使用。
- 6to4 位址：依 IPv4 位址自動產生部分 IPv6 位址，屬過渡技術，現今已較少使用。

#### 41. IPv6 與 IPv4 標頭異同
- IPv6 標頭較精簡、固定長度、沒有標頭檢查碼；IPv4 有可變長度選項。

#### 42. IPv4 選擇項與 IPv6 延伸標頭差異
- IPv4 Options 直接掛在基本標頭後，效率較差。
- IPv6 改用 Extension Header，需要時才附加，較模組化、效能較佳。
