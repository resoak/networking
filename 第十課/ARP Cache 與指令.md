# ARP Cache 與指令

## ARP Cache 在做什麼

如果每送一次資料都要重新廣播問一次 `MAC address`，效率會很差。

所以主機會把查過的 `IP ↔ MAC` 對照先記在 `ARP cache`。這樣之後再傳資料時，可以先查表，查得到就直接送，不必每次都重新做 ARP request。

## 為什麼需要快取

`ARP cache` 的主要用途有三個：

- **提升效率**，減少重複廣播。
- **減少等待時間**，讓資料更快送出。
- **降低區網負擔**，不要讓每台電腦一直收到不必要的 ARP 查詢。

## Dynamic 與 Static

### Dynamic 項目

- 由系統透過 ARP 自動學到。
- 平常最常看到的是這種。
- 會隨時間過期或被更新。

### Static 項目

- 由管理者手動加入。
- 不會像動態項目那樣自動學習或輕易變動。
- 常用在特定管理需求或實驗情境。

### 比較表

| 類型 | 來源 | 會不會自動更新 | 常見程度 |
|---|---|---|---|
| Dynamic | 系統透過 ARP 自動學到 | 會 | 很常見 |
| Static | 人工手動設定 | 通常不會 | 較少見 |

## 常見指令概念

課堂中主要用的是 `arp` 指令。不同作業系統畫面可能略有差異，但核心觀念接近。

| 指令 | 概念 | 常見用途 |
|---|---|---|
| `arp -a` | 查看目前 ARP cache | 看系統已經記住哪些 `IP ↔ MAC` |
| `arp -d` | 刪除 ARP 項目 | 強迫系統下次重新查詢 |
| `arp -s` | 手動加入靜態項目 | 建立固定對照 |

## Windows、macOS、Linux 的觀念整理

### Windows

- `arp -a`：列出快取。
- `arp -d <IP>`：刪除某一筆。
- `arp -d *`：清空目前 ARP cache。
- `arp -s <IP> <MAC>`：加入靜態項目。
- 某些刪除或新增操作常需要系統管理員權限。

### macOS

- 也能用 `arp -a` 查看快取。
- 刪除或新增時通常要配合管理權限。
- 白話上和 Windows 很接近，只是權限與顯示格式可能不同。
- 有些操作需要先用 `sudo`。

### Linux

- 傳統上也能看到 `arp -a` 這類操作。
- 刪除與新增靜態項目通常也需要較高權限。
- 不同發行版工具細節可能不同，但概念仍然是查看、刪除、加入對照。
- 在終端機實驗時，如果是持續 `ping`，可用 `Ctrl + C` 停止。

> 這一課的重點不是背每個系統的完整參數，而是知道這三種動作在做什麼。

## 課堂實作常見操作

老師示範的實驗邏輯大致是：

1. 先用 `arp -a` 看目前快取。
2. 用 `arp -d <IP>` 或 `arp -d *` 刪除一筆或整份快取。
3. 再去 `ping` 同網段主機或外部 IP。
4. 最後重新看 `arp -a`，觀察系統是學到目標主機 MAC，還是學到 Router MAC。

## 用 ping 觀察 ARP Cache

### 情境一，同網段主機

如果你先把快取清掉，再 `ping` 同網段主機，之後查看 `arp -a`，通常會看到：

- 新增了那台同網段主機的 `IP ↔ MAC` 對照。
- 代表系統先透過 ARP 找到它的 MAC，之後才把 `ping` 資料送出去。

### 情境二，不同網段主機

如果你 `ping 8.8.8.8` 這類外部位址，之後查看 `arp -a`，常看到的是：

- `default gateway` 的 MAC。
- 不一定會看到 `8.8.8.8` 的 MAC。

原因是外部主機不在本地區網，主機只需要先把資料交給 Router。

## 同網段與不同網段再整理一次

| 目標位置 | 先查誰的 MAC | 為什麼 |
|---|---|---|
| 同網段主機 | 目標主機 | 因為第二層 frame 可以直接送到對方 |
| 不同網段主機 | Default Gateway | 因為主機要先把資料交給 Router 轉送 |

## Wireshark 觀察 ARP

在 Wireshark 內可以觀察到這些現象：

- 先出現 `ARP request`。
- request 常是廣播，所以目的 MAC 常接近廣播位址概念。
- 接著出現 `ARP reply`，由目標主機回覆自己的 MAC。
- 完成後，後續的 `ping` 或其他資料封包才會持續送出。

這很適合驗證一件事：

> `ping` 之前，系統其實常常先偷偷做了 ARP。

## 靜態項目要注意什麼

靜態 ARP 雖然能減少重複查詢，但也有風險：

- 如果手動設定錯 MAC，資料可能送不到。
- 網路拓樸變動時，靜態資料不會自己修正。
- 所以大多數一般使用情境，還是讓系統用動態學習比較方便。

## 怎麼記

- `ARP cache` 是位址對照表。
- `arp -a` 看表。
- `arp -d` 刪表。
- `arp -s` 加靜態表。
- 同網段看目標主機，不同網段看 Router。
