# 快速記憶卡

## ARP / RARP

- `ARP`：`IP address → MAC address`。
- `RARP`：`MAC address → IP address`，較早期、較少見。
- 同網段時，ARP 直接找目標主機的 MAC。
- 不同網段時，ARP 先找 `default gateway` 的 MAC。

## ARP 流程

- 先查 `ARP cache`。
- 查不到才送 `ARP request`。
- `ARP request` 常用廣播。
- `ARP reply` 通常由目標主機回給詢問者。
- `ARP` 常見 EtherType：`0x0806`。
- `RARP` 歷史上常見 EtherType：`0x8035`。
- 成功後把 `IP ↔ MAC` 寫進 `ARP cache`。

## ARP Cache

- 目的：提升效率，減少重複廣播。
- `dynamic`：系統自動學到，會過期。
- `static`：人工設定，較固定。
- `arp -a`：看快取。
- `arp -d`：刪快取。
- `arp -d *`：整份 ARP cache 清空。
- `arp -s`：加靜態項目。

## ICMP

- `ICMP`：網路錯誤回報、控制與診斷。
- `ping`：用 `ICMP Echo Request / Reply` 測可達性。
- `ping` 適合先查第三層，不代表第 4 到第 7 層一定正常。
- `Type 8`：`Echo Request`。
- `Type 0`：`Echo Reply`。
- `Type 3`：`Destination Unreachable`。
- `Type 11`：`Time Exceeded`。

## ICMP Header

- `Type`：訊息大類。
- `Code`：細分原因。
- `Checksum`：檢查封包。
- `Identifier`：同一次 `ping` 常相同。
- `Sequence Number`：每一對 request / reply 對應同一序號。

## Destination Unreachable

- `Type 3 Code 0`：網路不可達。
- `Type 3 Code 1`：主機不可達。
- `Type 3 Code 2`：協定不可達。

## Traceroute

- 用來看封包經過哪些路由器。
- 核心依靠 `TTL` 與 `ICMP Time Exceeded`。

## Wireshark 觀察

- 先看有沒有 ARP request / reply。
- 再看 ICMP 的 `Type`、`Code`、`identifier`、`sequence number`。
- 同網段 `ping` 看得到目標主機 MAC。
- 不同網段 `ping` 常先看到 Router 的 MAC。

## 診斷順序

- `127.0.0.1`：測本機 TCP/IP。
- 本機網卡 IP：測網卡。
- Router IP：測區網出口。
- `8.8.8.8`：測對外連線。
- 網域名稱：測 DNS。
