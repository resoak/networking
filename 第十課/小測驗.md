# 小測驗

## 題目

1. `ARP` 的主要功能是什麼？
2. `ARP` 與 `RARP` 的方向有什麼差別？
3. 為什麼主機只知道 `IP address` 還不夠，還要知道 `MAC address`？
4. `ARP request` 和 `ARP reply` 的基本流程是什麼？
5. 為什麼 `ARP request` 常用廣播送出？
6. 主機在送資料前，為什麼會先查 `ARP cache`？
7. `dynamic` 與 `static` ARP 項目有什麼差別？
8. `arp -a`、`arp -d`、`arp -s` 分別在做什麼？
9. 如果目標主機和我在同網段，我會先查誰的 MAC？
10. 如果目標主機和我不同網段，我會先查誰的 MAC？
11. 為什麼 `ping 8.8.8.8` 之後，`arp -a` 常看到的是 Router 的 MAC？
12. `ICMP` 的主要角色是什麼？
13. `ping` 用到的是哪兩種 ICMP 訊息？
14. `Type 8` 與 `Type 0` 分別代表什麼？
15. `Identifier` 與 `Sequence Number` 在 `ping` 封包裡各有什麼用途？
16. `Destination Unreachable` 是哪一個 Type？為什麼還要再看 Code？
17. `Type 3 Code 0`、`Code 1`、`Code 2` 分別常表示什麼？
18. `Redirect`、`Time Exceeded`、`Parameter Problem` 各在說什麼狀況？
19. `traceroute` 為什麼能看出一路經過哪些 Router？
20. 如果 `ping 127.0.0.1` 不通，通常優先懷疑哪裡？
21. 如果 `ping` 本機網卡 IP 不通，通常優先懷疑哪裡？
22. 如果 `ping Router` 不通，但 `ping` 隔壁同網段主機有通，可能比較像哪裡有問題？
23. 如果外部 IP 不通，但 Router 有通，可能往哪個方向查？
24. 如果 `8.8.8.8` 可以通，但 `www.google.com` 不通，最可能是哪裡有問題？

## 答案

1. **把 `IP address` 解析成 `MAC address`**。
2. **ARP 是 `IP → MAC`；RARP 是 `MAC → IP`**。
3. **因為在區域網路中，第二層送 frame 時要填目的 MAC，不能只靠 IP**。
4. **先查 cache，查不到就廣播 ARP request，目標主機再回 ARP reply**。
5. **因為一開始不知道目標主機是哪一台，只能先在同網段問大家**。
6. **為了提升效率，避免每次都重新廣播**。
7. **dynamic 是系統自動學到、會過期；static 是人工設定、較固定**。
8. **`arp -a` 看快取；`arp -d` 刪除快取項目；`arp -s` 加入靜態項目**。
9. **查目標主機的 MAC**。
10. **查 `default gateway` 或 Router 的 MAC**。
11. **因為外部主機不在本地區網，主機先把資料交給 Router 轉送**。
12. **做網路錯誤回報、控制與診斷**。
13. **`Echo Request` 與 `Echo Reply`**。
14. **`Type 8 = Echo Request`；`Type 0 = Echo Reply`**。
15. **`Identifier` 用來識別同一次 ping；`Sequence Number` 用來區分每一對 request / reply**。
16. **`Type 3`；因為還要看 Code 才知道是網路、主機、還是協定不可達**。
17. **`Code 0 = 網路不可達`；`Code 1 = 主機不可達`；`Code 2 = 協定不可達`**。
18. **`Redirect` 是有更好的下一跳；`Time Exceeded` 是 TTL 用完；`Parameter Problem` 是封包欄位或格式可能有問題**。
19. **因為它利用逐步增加的 TTL，讓每一跳 Router 回 `ICMP Time Exceeded`**。
20. **本機 TCP/IP 軟體堆疊**。
21. **網卡或本機網路介面設定**。
22. **Router 或本機到 Router 的那段區網**。
23. **對外路由、ISP 或 Internet 連線**。
24. **DNS 名稱解析**。
