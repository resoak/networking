# 網路連線診斷流程

## 為什麼要有固定流程

網路不通時，如果一開始就直接猜「是不是 DNS 壞了」或「是不是 ISP 壞了」，很容易亂掉。

老師在錄音 16 提供了一個很好背、也很好用的基本排查順序。它的好處是能把問題一步一步縮小。

## 核心順序

> `127.0.0.1 → 本機網卡 IP → Default Gateway / Router → 外部 IP → 網域名稱`

## 第一步，ping `127.0.0.1`

### 這在檢查什麼

- 檢查本機 `TCP/IP` 軟體堆疊。
- 這個封包只在自己電腦內部回圈，不會送到網卡。

### 如果這一步不通

- 問題比較偏向本機系統或 TCP/IP 軟體設定。
- 連最基本的本機網路堆疊都異常，後面就不用急著查外網。

## 第二步，ping 自己網卡的 IP

### 這在檢查什麼

- 檢查自己的網卡是否正常工作。
- 主要檢查本機 IP 設定、介面綁定與本機網路堆疊是否大致正常。

### 如果這一步不通

- 可能是網卡、驅動程式、介面設定有問題。
- 即使第一步正常，也不代表網卡一定沒事。
- 但要注意：這一步有通，也不等於實體網卡、網路線或 Wi-Fi 連線就一定完全沒問題。

## 第三步，ping Default Gateway 或 Router 的 IP

### 這在檢查什麼

- 檢查本機到本地網路出口是否正常。
- 有線環境通常是 Router。
- 無線環境常是無線 AP 或家用分享器。

### 如果這一步不通

可能的方向：

- 網路線問題
- Wi-Fi 連線問題
- Router 或 AP 問題
- 本機與閘道之間的區網有問題

老師也提到，如果能 `ping` 到隔壁主機，卻 `ping` 不到 Router，就比較像是 Router 本身有問題。

## 第四步，ping 外部 IP，例如 `8.8.8.8`

### 這在檢查什麼

- 本機是否能成功往外送到 Internet。
- 本地區網以外的路由是否正常。
- ISP 連線是否大致正常。

### 如果這一步不通

可能代表：

- Router 雖然活著，但對外沒通。
- ISP 線路異常。
- 中間路由有問題。

老師的白話做法是，再換一個外部 IP 試試看。如果多個外部 IP 都不通，就比較可能是 ISP 或對外路由問題。

## 第五步，ping 網域名稱，例如 `www.google.com`

### 這在檢查什麼

- 是否能做 **DNS 名稱解析**。
- 系統是否能把網域名稱轉成對應的 IP。

### 如果外部 IP 通，但名稱不通

這很常代表：

- DNS server 有問題。
- DNS 設定錯誤。
- 名稱解析被擋掉。

這時候就不是先去懷疑路由，而是優先查 DNS。

## 整體判讀表

| 測試步驟 | 如果不通，優先懷疑 |
|---|---|
| `ping 127.0.0.1` | 本機 TCP/IP 軟體堆疊 |
| `ping` 本機網卡 IP | 網卡或本機介面設定 |
| `ping` Router IP | 區網、網路線、Wi-Fi、Router / AP |
| `ping 8.8.8.8` | 對外路由、ISP、外網連線 |
| `ping www.google.com` | DNS 解析 |

## 為什麼這順序很重要

這個順序是從**最靠近自己**的問題往**最外面**查。

- 先確定自己電腦正常。
- 再確定自己網卡正常。
- 再確定本地出口正常。
- 再確定 Internet 路徑正常。
- 最後再檢查名稱解析。

這樣不容易跳步，也比較不會把小問題誤判成大問題。

另外，`ping` 之所以常被拿來當第一線工具，就是因為它先幫我們確認第三層的基本可達性，能暫時避開很多傳輸層、應用層服務造成的干擾。

## 搭配 ARP 的理解

在第 2 步到第 4 步之間，常常會順便看到 ARP 的效果：

- `ping` 同網段 IP 之前，系統可能先做 ARP 找該主機的 MAC。
- `ping` 外部 IP 之前，系統可能先做 ARP 找 `default gateway` 的 MAC。

所以排錯時，如果 `ping` 異常，也可以配合 `arp -a` 看系統到底學到了誰的 MAC。

## 搭配 Wireshark 的理解

如果想再看細一點，可以抓封包觀察：

- 有沒有先出現 ARP request / reply。
- `ping` 是否真的送出 `ICMP Echo Request`。
- 有沒有收到 `Echo Reply`。
- 如果沒收到回應，有沒有看到 `Destination Unreachable` 或 `Time Exceeded`。

## 實務提醒

- Windows 內建的自動疑難排解能幫忙，但通常比較慢。
- 真正工作現場更重要的是你自己會照流程查。
- 只要順序熟，很多問題其實很快就能定位到大方向。

## 一句話總結

不能上網時，不要亂猜。

先測自己，再測網卡，再測 Router，再測外部 IP，最後再看 DNS。
