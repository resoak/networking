# 入侵偵測系統 IDS

## IDS 是什麼？

IDS（Intrusion Detection System，入侵偵測系統）用來監控網路或主機是否出現可疑行為、攻擊特徵或違反安全規則的活動。

## IDS 與防火牆差異

| 工具  | 主要任務       | 比喻  |
| --- | ---------- | --- |
| 防火牆 | 依規則允許或拒絕流量 | 門禁  |
| IDS | 偵測可疑行為並告警  | 監視器 |

防火牆不一定看得出所有攻擊；IDS 則負責發現異常或攻擊跡象，但不一定會主動阻擋。

## 依部署位置分類

### NIDS：網路型 IDS

部署在網路節點或網段上，監看流量。

優點：

- 可以監控多台主機的流量。
- 適合觀察整體網路攻擊跡象。

限制：

- 高流量時可能漏看。
- 加密流量不容易分析內容。
- 部署位置會影響可見範圍。

### HIDS：主機型 IDS

安裝在單一主機上，監看該主機的系統記錄、檔案、程序與登入行為。

優點：

- 能看到主機內部狀態。
- 適合保護重要伺服器。

限制：

- 每台主機都要部署與維護。
- 主機被攻陷時，IDS 也可能被關閉或竄改。

## 偵測方式

- **特徵式偵測**：比對已知攻擊特徵。
- **異常式偵測**：找出和平常不同的行為。
- **日誌分析**：從系統與服務紀錄中找可疑事件。

## 管理注意事項

IDS 不是裝了就結束，還需要：

- 更新規則或特徵碼。
- 調整告警門檻。
- 判讀誤報與漏報。
- 與防火牆、SIEM、備份、事件應變流程搭配。

## 怎麼記

> 防火牆負責「擋門口」，IDS 負責「看有沒有可疑行為」。
