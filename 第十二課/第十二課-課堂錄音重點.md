## 課堂主軸

這一課的核心是：**網路安全不是單一工具，而是一組保護資料、身分與服務可用性的策略。**

課堂用企業資料外洩、勒索、假冒身分、攻擊網站等例子提醒：只要資料會透過網路傳送，就可能面臨竊聽、竄改、偽冒或中斷服務的風險。

## 1. 資訊安全與 CIA

資訊安全的三大目標是：

| 目標  | 英文              | 意思           | 例子           |
| --- | --------------- | ------------ | ------------ |
| 機密性 | Confidentiality | 只有授權者能看到資料   | 加密、權限控管      |
| 完整性 | Integrity       | 資料沒有被未授權竄改   | 雜湊、數位簽章      |
| 可用性 | Availability    | 需要時系統能正常提供服務 | 備份、防 DDoS、容錯 |

老師用「保險箱」的概念說明：資料不只要藏得住，也要在合法使用者需要時拿得到，而且不能被偷偷換掉。

## 2. 常見安全問題

網路安全問題常見在：

- **實體層面**：設備、機房、儲存媒體被破壞或偷走。
- **人員層面**：密碼外洩、社交工程、內部人員誤用權限。
- **系統層面**：作業系統、應用程式、資料庫漏洞。
- **網路通訊層面**：封包被竊聽、竄改、偽冒或阻斷。

常見威脅包含惡意程式、網路釣魚、密碼攻擊、DDoS、中間人攻擊、DNS 偽造、SQL injection、XSS、零日攻擊與供應鏈攻擊等。

## 3. 常見防護策略

課堂反覆出現的基本策略有四個：

1. **防火牆**：控管哪些封包或連線可以通過。
2. **加密**：讓未授權者即使取得資料也看不懂。
3. **認證**：確認對方是不是真正的使用者或伺服器。
4. **入侵偵測系統 IDS**：監控可疑活動並提出警告。

補充：課本習題也把 **數位簽章** 列為常見策略，因為它能處理來源確認、完整性與不可否認性。

## 4. 防火牆

防火牆通常放在不同信任程度的網路中間，例如公司內部網路與 Internet 之間，也可以放在部門之間或個人電腦上。

課堂提到兩種基本類型：

- **封包過濾式防火牆**：看來源 IP、目的 IP、協定、連接埠等欄位，速度快但不深入理解應用層內容。
- **應用閘道式／代理式防火牆**：在應用層代替使用者連外，可以檢查內容與記錄行為，但成本與負擔較高。

## 5. 加密與金鑰

加密的基本流程：

```text
明文 + 加密演算法 + 金鑰 → 密文
密文 + 解密演算法 + 金鑰 → 明文
```

重點不是「演算法完全不能被知道」，而是即使演算法公開，只要金鑰安全，攻擊者也很難在合理時間內破解。

## 6. 秘密金鑰與公開金鑰

| 類型       | 又稱     | 金鑰用法        | 優點            | 問題     |
| -------- | ------ | ----------- | ------------- | ------ |
| 秘密金鑰密碼系統 | 對稱式加密  | 加密與解密用同一把金鑰 | 速度快，適合大量資料    | 金鑰配送困難 |
| 公開金鑰密碼系統 | 非對稱式加密 | 公鑰公開、私鑰保密   | 可解決金鑰配送，也能做簽章 | 速度較慢   |

實務上像 HTTPS/TLS 常採用混合方式：先用公開金鑰方法建立安全通道或交換暫時金鑰，再用速度較快的對稱式加密傳大量資料。

## 7. 雜湊函數與完整性

雜湊函數會把任意長度資料轉成固定長度的摘要。它常用來檢查資料是否被改過：

1. 發送端對資料算出摘要。
2. 接收端也對收到的資料算摘要。
3. 兩邊摘要相同，代表資料大致可信；不同則代表資料可能遭竄改或傳輸錯誤。

好用的雜湊函數應具有：難以反推原文、輸入一點點改變就讓摘要大幅改變、碰撞機率低。

## 8. 認證、數位簽章與憑證

**認證** 是確認身分；**加密** 則是保護內容不要被偷看。兩者不同但常一起使用。

數位簽章的觀念：

- 發送者用自己的 **私鑰** 對資料或資料摘要簽章。
- 接收者用發送者的 **公鑰** 驗證。
- 可確認來源、資料完整性，並提供不可否認性。

數位憑證則負責回答：「這把公鑰真的屬於這個人或網站嗎？」憑證由可信任的 CA 簽發，是 HTTPS、線上銀行與電子化政府服務的重要基礎。

## 9. 入侵偵測系統 IDS

IDS 不是防火牆。防火牆偏向「依規則擋或放行」，IDS 偏向「監控與發現異常」。

- **NIDS（網路型）**：部署在網路中，監看流量與攻擊特徵。
- **HIDS（主機型）**：部署在單一主機，監看系統記錄、檔案與程序行為。

IDS 需要規則、特徵碼或異常偵測能力，也需要管理者判讀告警。

## 怎麼記

- CIA：**機密不外洩、完整不被改、可用不中斷**。
- 防火牆：像大樓門禁，決定誰能進出。
- 加密：像把內容鎖起來，沒金鑰看不懂。
- 雜湊：像資料指紋，用來檢查有沒有變。
- 數位簽章：像電子簽名，確認誰簽的、內容有沒有被改。
- 憑證：像身分證明，證明公鑰屬於誰。
- IDS：像監視器，負責發現可疑行為。
