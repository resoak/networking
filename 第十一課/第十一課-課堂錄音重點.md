## 導覽

- 這一課從「路由器到底是什麼」開始。
- 老師先用家用 Wi-Fi、寬頻撥號、Router 內外網的例子，讓大家知道：連上 Wi-Fi 不等於已經連到 Internet。
- 接著進入正式的 routing 觀念：
  - Router 會連接不同網路。
  - 封包到達 Router 後，Router 會看目的地 `IP address`。
  - Router 再根據 `routing table` 決定要往哪個 interface 或下一跳送。
- 後面再比較路由資訊從哪裡來：直接連接、人工靜態設定、或由動態路由協定交換。

## 課堂主軸

老師這一課的主軸是：**封包不是一次就從你的電腦飛到目的地主機，而是經過一台一台 Router 往外轉送**。

Router 的工作不是單純「讓大家上網」而已。更精確地說，它會連接不同的第三層網路，並依照目的地位址與路由資訊，判斷封包下一步該往哪裡走。

因此這一課一直圍繞三個問題：

1. Router 怎麼知道封包該往哪裡送？
2. `routing table` 裡的資料是怎麼來的？
3. 當網路變大、路徑變多、狀態會改變時，要怎麼更新這些資訊？

## Router 與路徑選擇

錄音 17 的開場先把家用 Router 拆成內部與外部來看。內部是 `LAN`，例如家裡手機、筆電連到 Router，常見 private IP 可能是 `192.168.x.x`；外部是 `WAN` 或 ISP 端，Internet 上其他主機真正能看到的是 Router 對外的 public IP。

所以如果別人問「你家的 IP」，不能只回答內網的 `192.168.x.x`。這比較像說「我在家裡的臥室」，但外面的人真正需要的是能找到你家大門的地址，也就是對外的 public IP。

老師用交通導航作比喻：如果你從公司回家，不只要知道目的地，也要知道哪條路比較適合。網路中的封包也是一樣，Router 要根據某些資訊判斷下一段路。

這裡的「距離」不一定是實際的物理長度。課堂提到，Routing metric 可能考慮：

- 經過幾個 Router，也就是 `hop count`。
- 線路速度快慢。
- 線路穩定度。
- 是否常重傳或容易出錯。
- 成本或費用。
- 目前是否壅塞。

所以「最好路徑」不一定只是最短，也可能是最快、最穩、最便宜，或依協定設計出來的綜合分數最低。

## Routing Table 的角色

`routing table` 可以想成 Router 的地圖或導航表。當封包進入 Router 時，Router 會看目的地 `IP address`，再查路由表決定：

- 目的網路是哪一個。
- 這筆路由要從哪個 interface 送出。
- 下一跳 Router 是誰。
- 這條路由是直接連接、靜態設定，還是動態學到。

錄音 19 用多個網路範例說明：如果 Router 只知道 `192.168.0.0`、`10.0.0.0`、`125.0.0.0`，但不知道 `174.16.0.0`，那封包要去 `174.16.0.0` 時就可能被丟掉。這時候就需要額外加入 static route，或讓動態路由協定自動學到。

## 直接連接、靜態路由、動態路由

課堂把 routing table 的來源整理成三種：

| 類型 | 來源 | 特徵 |
|---|---|---|
| Directly connected route | Router interface 直接連到的網路 | 設定 interface 的 `IP address` 與 `subnet mask` 後自動產生 |
| Static route | 網管人員手動設定 | 簡單、可控，但網路變動時不會自動修正 |
| Dynamic route | 相鄰 Router 透過 routing protocol 交換資訊 | 適合較大或常變動的網路，但需要額外協定與更新成本 |

老師特別提醒靜態路由可能有方向問題。假設 `192.168.0.0` 能透過下一跳到 `174.16.0.0`，不代表 `174.16.0.0` 也知道怎麼回到 `192.168.0.0`。如果要雙向溝通，回程路由也要存在。

## Distance Vector 路由

錄音 18 主要補強 `Distance Vector` 的概念。

Distance Vector 可以白話理解成：每台 Router 定期和相鄰 Router 交換自己知道的路由資訊。收到別人的資訊後，再加上「從我到鄰居」這段距離，更新自己的 routing table。

它的優點是：

- 概念簡單。
- 路由器負擔相對低。
- 小型網路中容易運作。

它的限制是：

- 大型網路中交換資訊的成本會變高。
- 每台 Router 知道的通常不是完整拓撲，而是從鄰居聽來的距離資訊。
- 網路狀態改變時，更新需要時間傳開，收斂可能比較慢。

課堂中提到 `RIP` 可作為 Distance Vector 類型的代表。

## Link State 與 OSPF

`Link State` 的想法和 Distance Vector 不太一樣。它不是只聽鄰居說「到某地多遠」，而是讓 Router 分享自己的連線狀態，例如：

- 我連到哪些鄰居。
- 連線成本是多少。
- 連線是否健康。

這些資訊會透過 flooding 傳播，讓同一區域內的 Router 逐漸掌握較完整的拓撲，再用 Dijkstra 最短路徑演算法計算到各目的地的較佳路徑。

課堂中提到 `OSPF` 是重要的 Link State 路由協定。因為大型網路如果全部一起交換完整狀態會太複雜，所以 `OSPF` 會把網路切成不同 area，並透過 backbone area 連接，降低規模變大時的管理與計算負擔。

## Hybrid 與 EIGRP

老師最後把三種做法整理起來：

- `Distance Vector`：較簡單，主要和鄰居交換距離或路由表資訊。
- `Link State`：資訊較完整，但一開始建立與計算會比較複雜。
- `Hybrid`：試著把兩者的優點結合起來。

課堂中提到 `EIGRP` 可以視為混合式路由協定的代表。它不是單純只用 Distance Vector，也不是完全照 Link State 的方式，而是結合部分特性，讓路由更新與路徑選擇更有效率。

## Dijkstra 與 Bellman-Ford 補充

錄音 20 把兩種演算法直覺再講清楚。

`Dijkstra` 可以想成每台 Router 已經掌握一張完整地圖，接著從自己出發，每次挑目前距離最小、還沒確認的節點，把它加入最短路徑集合，再用它更新周圍節點距離。它適合 link cost 都是非負數的情況。

`Bellman-Ford` 的直覺比較像 Distance Vector：我問每個鄰居「你到目的地要花多少」，再加上我到那個鄰居的成本，挑總成本最低的那條路。這種做法簡單，但如果網路狀態變動傳得慢，就可能造成錯誤資訊互相傳遞。

錄音 20 也補充：路由協定使用的「距離」不一定是地理距離。早期或較簡單的協定可能只看 `hop count`，也就是經過幾台 Router；但實際選路還可能考慮 `delay`、`bandwidth`、`load`、可靠度與費用等因素。實務協定會把多種因素整理成 metric 或權重，讓 Router 可以用數值比較哪一條路比較適合。

老師用導航比喻說明：開車從台北到高雄時，最好路線不一定是地圖上最短，而可能是最快、最少塞車、最穩定的路。網路中的 routing 也是類似概念，Router 不是只看「遠近」，而是依照協定定義的 metric 選出較合適的下一跳。

錄音 20 對三類動態路由的整理如下：

| 類型 | 白話理解 | 特徵 |
|---|---|---|
| Distance Vector | 問鄰居到目的地多遠 | 設定簡單、資源需求低，但收斂較慢 |
| Link State | 每台 Router 掌握較完整地圖後自己算路 | 資訊完整、計算精準，但初期建立較複雜 |
| Hybrid | 混合兩者優點 | 想兼顧更新效率與路徑品質 |

錄音中還用「水流」比喻 Dijkstra：從起點倒水，水會沿著不同長度的連線流出去，越短的路徑越早到達下一個節點；當水先到達某個節點，就代表目前找到到該節點的最短距離。這也說明為什麼 Dijkstra 不適合負權重：如果某條邊的成本可以是負數，已經確認的最短路徑可能又被後面路徑修正，演算法假設就被破壞。

## 路由迴圈與收斂

Distance Vector 類型協定可能出現 routing loop。當某段網路已經斷線，但 Router 之間還沒同步完成時，彼此可能誤以為對方還有路，造成路由資訊或封包在幾台 Router 間繞圈。

這會導致 `count to infinity`：距離數字一直增加，直到達到協定定義的最大值才被視為不可達。課堂提到的改善方式包含最大 hop count、split horizon、poison reverse、hold-down timer 與 triggered update。

## AS、IGP、EGP 與 BGP

錄音 21 從大型網際網路的角度補充：不可能讓每台 Router 都保存全世界所有路由細節，所以會把網路切成 `Autonomous System`。

同一個 AS 內部使用的路由協定稱為 `IGP`，例如 `RIP`、`OSPF`、`EIGRP`。不同 AS 之間使用的路由協定稱為 `EGP`，現代 Internet 的代表是 `BGP`。

`BGP` 會記錄路徑經過哪些 AS，也就是 `AS path`。如果同一個 AS 在路徑中重複出現，就可以判斷可能有迴圈。此外，BGP 也常依政策與管理需求選路，不只是單純比距離。

錄音 21 也補充了幾個協定定位：

- `RIP` 屬於 IGP，常用 hop count 作為 metric，早期常見但受限於網路規模與收斂速度。
- `OSPF` 屬於 IGP，採用 Link State 思想，會利用 area 概念降低大型網路中的 flooding 與計算負擔。
- 早期 `EGP` 協定已不太適合現代 Internet，現在跨 AS 路由主要看 `BGP`。
- `BGP` 透過 TCP 傳送路由資訊，並用 AS path 來輔助避免 routing loop。

## AD、Cost、容錯與負載平衡

錄音 21 的台北、台中、高雄範例說明了多條路徑存在時 Router 怎麼選路。

如果同一目的地有不同來源的路由，Router 會先看 `Administrative Distance`。AD 數字越小，代表優先順序越高。若 AD 相同，再比較協定內部的 `cost`。例如 OSPF 常見 AD 是 `110`；如果手動設定 static route 的 AD 為 `50`，它會優先於 OSPF。

多拉幾條專線可以提供容錯：某條線斷掉時，還能改走另一條。若兩條路 cost 相同，也可能做負載平衡，把流量分散到不同路徑。

錄音 21 的影片範例把概念講得更具體：台北、台中、高雄三台 Router 之間共有六個網段。如果全部靠 static route，每台 Router 都要手動設定多筆路由；只要少設一筆或設錯一筆，就可能造成網路不通。啟動 `OSPF` 後，Router 會自動交換自己知道的網段，讓三台 Router 都學到完整路由。

在 OSPF 範例中，路由表前面的代號也有意義：`C` 代表 directly connected route，是 Router 自己 interface 直接連到的網段；`O` 代表透過 OSPF 學到的動態路由。這有助於看懂路由表中每一筆路由的來源。

錄音 21 對 cost 的例子：若台北到某目的網段有兩條路，一條 cost 是 `65`，另一條是 `129`，在 AD 相同時 Router 會選 `65`。OSPF cost 可依頻寬計算，影片中以 `10^5 / bandwidth` 的方式示範：`1544K` 專線約得到 cost `64`，`100M` 網段約得到 cost `1`，路徑總 cost 則是沿路各段相加。

錄音 21 也說明靜態與動態路由可以混用：平常大部分網段交給 OSPF 自動維護，但若某台重要主機因安全考量希望走指定路徑，可以新增一條 AD 較小的 static route。例子中 static route 設成 AD `50`，因此會優先於 OSPF 的 AD `110`；如果指定路徑斷線，才回到其他可用路徑。

## 路由診斷指令

錄音 22 補充幾個實用指令：

- `ping`：測試目的主機是否有回應，但有些設備會擋 ICMP。
- `tracert`：透過 TTL 逐步增加，列出到目的地中間經過哪些 Router。
- `arp -a`：查看本機 ARP cache，也就是 IP address 與 MAC address 的對應。
- `route print`：查看本機 routing table。
- `pathping`：類似加強版 tracert，會統計路徑上的封包遺失比例。

錄音 22 對 `tracert` 的說明比較細：它會先送 TTL=1 的封包，讓第一台 Router 回應；再送 TTL=2、TTL=3，逐步把中間節點列出來。因此 `tracert` 適合回答「封包到底走哪一條路」。如果中間某些節點不回應，畫面可能出現等待或星號，這不一定代表路徑中斷，也可能只是設備不回 ICMP。

課堂中以學校網站為例，路徑可能先到實驗室 gateway，再到大樓或校園網路設備，甚至因網站放在外部代管或防護設備後面而先繞到外部節點。這提醒我們：看到路徑「跑出去」不一定是錯，可能是因為服務實際架設位置或安全防護架構不同。

`arp -a` 可以用來看 IP address 與 MAC address 的對應；必要時也可以設定靜態 ARP 項目，但課堂重點是先看懂 ARP cache。`route print` 則可以看本機 routing table，理解本機要往不同目的網段時會走哪個 gateway 或 interface。

`pathping` 是 `tracert` 的進階版。一般 `tracert` 主要列出路徑，`pathping` 會花比較久時間做多次測試，最後統計每一段路徑的封包遺失比例。當你懷疑某段網路不穩、丟包嚴重時，`pathping` 比單純看路徑更有幫助。

## 怎麼記

- Router：連接不同網路，替封包找下一步。
- Routing：從來源到目的地之間選路。
- Routing table：Router 的地圖或導航表。
- Directly connected：interface 直接連到的網路。
- Static route：人工寫進去的路。
- Dynamic route：Router 彼此交換資訊後學到的路。
- Distance Vector：跟鄰居交換距離或路由表。
- Link State：分享連線狀態，用 Dijkstra 算路徑。
- OSPF：常見 Link State 協定，會用 area 降低複雜度。
- EIGRP：混合式協定，結合不同做法的優點。
- AS：可獨立管理的大型路由區塊。
- IGP：AS 內部使用的路由協定。
- EGP/BGP：AS 之間使用的路由協定，現代代表是 BGP。
- AD：比較路由來源優先權，越小越優先。
- Cost：同一協定內比較路徑成本，越小越優先。
- Tracert：追蹤封包經過的 Router。
