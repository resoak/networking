# Router 與 Routing

## Router 是什麼

`Router` 是用來連接不同網路的設備。它會根據封包裡的目的地 `IP address`，決定封包下一步要往哪裡送。

白話來說：

> Router 就像網路世界的轉運站，封包到它這裡後，它要判斷下一站是哪裡。

## 家用 Router 的例子

錄音一開始用家用 Wi-Fi 來建立直覺。

你手機顯示 Wi-Fi 已連線，只代表你連到家裡的 Router。可是 Router 外面還要連到 ISP、Internet、遠端伺服器。如果外面那段出問題，你仍然可能「Wi-Fi 已連線，但不能上網」。

所以可以分成兩段看：

- **家裡內部**：手機、筆電連到家用 Router。
- **家裡外部**：Router 再連到 ISP 與 Internet。

## Router 的內部與外部位址

錄音 17 還特別提醒：家用 Router 不是只有一個「位置」。它通常至少有內部與外部兩個角度。

| 角度 | 常見說法 | 白話理解 |
|---|---|---|
| 內部 | `LAN`、內網、私有網路 | 家裡手機、筆電、桌機看到的 Router |
| 外部 | `WAN`、外網、ISP 端 | Internet 上其他人看到的 Router |

在家裡內部，Router 常見位址可能像 `192.168.1.1`，而手機或電腦也可能拿到 `192.168.x.x` 這類位址。這些通常是 private IP，只在家裡或同一個內部網路裡有意義。

但外面的 Internet 主機要找你時，不會用你家裡的 `192.168.x.x`。它們看到的是 Router 對外、由 ISP 提供或分配的 public IP。

可以用錄音裡的比喻來記：

- `192.168.x.x` 像是「我在家裡的臥室」。
- public IP 像是「外面的人要找到你家大門的地址」。
- 快遞員要送東西到你家，不能只知道你在臥室；他需要能從外面找到你家的地址。

這也解釋了為什麼 Wi-Fi 圖示亮著還是可能不能上網：你可能只成功連到內部 LAN，但 Router 的 WAN 或 ISP 連線出了問題。

這也是為什麼家用 Router 常會牽涉三種密碼或設定：

| 類型 | 用途 |
|---|---|
| Wi-Fi 密碼 | 讓設備連進家裡無線網路 |
| 寬頻帳號密碼 | 讓 Router 代表使用者對外連線或撥號 |
| 管理密碼 | 登入 Router 管理介面修改設定 |

## Routing 是什麼

`Routing` 是替封包選路的過程。

封包從你的電腦送到遠端主機時，通常不是一次到達，而是：

1. 先到本地 Router。
2. 再到下一台 Router。
3. 一台一台往外轉送。
4. 最後到達目的地所在的網路。

Router 每次都要決定：

- 這個封包的目的地在哪裡？
- 我知道哪條路可以到那裡嗎？
- 下一跳要送給哪台 Router？
- 要從哪個 interface 送出去？

## 為什麼不能隨便轉送

如果 Router 只知道目的地 IP，卻不知道路要怎麼走，就不能可靠地把封包送到目的地。

這就像你知道朋友住在某個城市，但不知道要走哪條高速公路、轉哪個交流道。你需要地圖或導航，Router 也需要 [[Routing Table 路由表]]。

## Routing metric

課堂提到，Router 選路時會有「距離」或成本的概念，但這個距離不一定是物理距離。

可能考慮的因素包括：

- 經過幾個 Router，也就是 `hop count`。
- 線路速度。
- 線路穩定度。
- 錯誤率或重傳情況。
- 成本或費用。
- 壅塞程度。

所以「最佳路徑」不一定等於地理上最短，而是依照協定定義出來的 metric，選出最適合的路徑。

## 一句話記法

- `Router`：連接不同網路。
- `Routing`：替封包找下一站。
- `Metric`：衡量哪條路比較好的分數或成本。
- `LAN/WAN`：內部家用網路與外部 Internet 連線是兩個不同角度。
- `192.168.x.x`：常見 private IP，通常只在內部網路有意義。
