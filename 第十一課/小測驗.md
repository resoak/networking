# 小測驗

## 題目

1. Router 的主要工作是什麼？
2. 為什麼「Wi-Fi 已連線」不一定代表可以上 Internet？
3. `LAN` 和 `WAN` 在家用 Router 的例子中分別代表什麼？
4. 為什麼 `192.168.x.x` 這類位址通常不能直接當作外面 Internet 主機找你的地址？
5. `routing table` 的作用是什麼？
6. Routing table 的三種主要來源是什麼？
7. Directly connected route 是什麼？
8. Static route 和 dynamic route 最大差別是什麼？
9. 為什麼 static route 要注意回程路由？
10. `Distance Vector` 的基本想法是什麼？
11. `Link State` 的基本想法是什麼？
12. `OSPF` 為什麼會使用 area 的概念？
13. `EIGRP` 在這一課被歸在哪一類路由協定？
14. Routing metric 可能考慮哪些因素？請至少舉三個。
15. `AS` 是什麼？為什麼大型網路需要 AS？
16. `IGP` 和 `EGP` 的差別是什麼？`BGP` 屬於哪一類？
17. Router 選路時，`AD` 和 `cost` 通常哪一個先比較？
18. 什麼是 routing loop？Distance Vector 為什麼可能發生 count to infinity？
19. `tracert` 大致如何找出中間經過的 Router？
20. `ping`、`arp -a`、`route print`、`pathping` 各自主要用來看什麼？

## 答案

1. **連接不同網路並轉送封包**。Router 會依目的地 IP 與 routing table 決定下一跳或出口 interface。
2. **Wi-Fi 已連線只代表連到本地 Router**。Router 外面到 ISP 或 Internet 的路徑仍可能故障。
3. **`LAN` 是家裡內部網路，`WAN` 是 Router 對外連到 ISP / Internet 的那一側**。
4. **`192.168.x.x` 通常是 private IP，只在內部網路有意義**。外面 Internet 主機要找你時，通常看到的是 Router 對外的 public IP。
5. **告訴 Router 封包要往哪裡送**。它像地圖或導航，記錄目的網路、出口 interface、下一跳等資訊。
6. **Directly connected route、static route、dynamic route**。
7. **Router interface 直接連到的網路**。設定 interface 的 IP 與 subnet mask 後，Router 會自動產生這類路由。
8. **Static route 是人工設定，dynamic route 是 Router 透過 routing protocol 自動交換學到**。
9. **去程有路不代表回程有路**。如果對方網路不知道怎麼回到來源網路，雙向溝通仍會失敗。
10. **和相鄰 Router 交換距離或路由表資訊**，再更新自己的 routing table。`RIP` 是代表例子。
11. **分享連線狀態並建立較完整拓撲**，再用 Dijkstra 類型演算法計算較佳路徑。
12. **降低大型網路的 flooding 與計算複雜度**。Area 可以讓 Router 主要處理區域內資訊，再透過 backbone area 連接不同區域。
13. **Hybrid routing protocol**，也就是混合式路由協定。
14. 可以包含 **`hop count`、速度、穩定度、成本、壅塞程度、錯誤率或重傳情況**。
15. **`AS` 是由同一組織或同一管理政策控制的網路區塊**。大型網路用 AS 分層管理，避免每台 Router 都保存全世界所有細節。
16. **`IGP` 用在同一個 AS 內部，`EGP` 用在不同 AS 之間**。`BGP` 是現代主要的 EGP。
17. **通常先比 AD，再比 cost**。AD 比較路由來源優先權；cost 比較同一協定內的路徑成本。
18. **Routing loop 是封包或路由資訊在 Router 間繞圈**。Distance Vector 依賴鄰居資訊，當斷線消息傳得慢時，Router 可能互相誤信對方還有路，距離數字一直增加到最大值，形成 count to infinity。
19. **`tracert` 透過逐步增加 TTL 來找路徑**。TTL=1 找第一跳，TTL=2 找第二跳，依序列出中間節點。
20. **`ping` 看目的主機是否回應；`arp -a` 看 IP 與 MAC 對應；`route print` 看本機 routing table；`pathping` 追蹤路徑並統計丟包比例**。
