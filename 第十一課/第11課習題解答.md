# 第 11 課課後習題解答

> 主題：尋徑協定（Routing Protocol）  
> 依使用者提供的第 11 課課本習題照片整理。

---

## 一、填充題

1. 綜合性路徑距離計量所考慮的成分一般包含：**頻寬、負載、延遲、可靠度、路程計數、時脈計數與成本**。

2. 尋徑協定依其是否自動更新，可區分為：**靜態尋徑協定**與**動態尋徑協定**。

3. RIP 協定以 **路程計數（Hop Count）** 作為路徑距離計量的依據。

4. 動態尋徑協定依交換訊息機制的不同，可分為：**距離向量尋徑**、**連線狀態尋徑**，以及兩者結合的**混合式尋徑**。

5. 尋徑發生迴圈問題時，可用的解決方式有：**定義最大數值**、**分離界線（Split Horizon）**、**保持計時（Hold-Down Timer）**與**觸發更新（Triggered Update）**。

6. 尋徑協定依其在自律系統內外的運作，可區分為：**內部尋徑協定（Interior Routing Protocol）**與**外部尋徑協定（Exterior Routing Protocol）**。

7. 常見在自律系統內部的尋徑協定例子有：**RIP** 與 **OSPF**；外部的尋徑協定例子有：**EGP** 與 **BGP**。

8. RIP 每隔 **30 秒** 對相鄰路由器執行動態路徑資料交換；距離的最大數設為 **15**（補充：**16 代表不可達**）；路徑資料超過 **180 秒** 未更新則視為無效；超過 **240 秒** 未更新則將此資料刪除。

## 三、問答題

### 1. 請說明各項英文全名

| 縮寫 | 英文全名 | 中文意義 |
|---|---|---|
| RIP | Routing Information Protocol | 路由資訊協定 |
| OSPF | Open Shortest Path First | 開放式最短路徑優先協定 |
| IGRP | Interior Gateway Routing Protocol | 內部閘道路由協定 |
| EIGRP | Enhanced Interior Gateway Routing Protocol | 增強型內部閘道路由協定 |
| SPF | Shortest Path First | 最短路徑優先 |
| LSA | Link-State Advertisement | 連線狀態宣告／廣告 |
| IS-IS | Intermediate System to Intermediate System | 中介系統對中介系統協定 |
| AS | Autonomous System | 自律系統 |
| BGP | Border Gateway Protocol | 邊界閘道協定 |
| EGP | Exterior Gateway Protocol | 外部閘道協定 |

### 2. 比較 Router、Routing、Routing Protocol、Route 與 Routing Table

- **Router（路由器）**：負責連接不同網路，並依據目的網路位址轉送封包的設備。
- **Routing（尋徑／路由選擇）**：決定封包應該走哪一條路徑到達目的地的過程。
- **Routing Protocol（尋徑協定）**：路由器之間交換路由資訊、計算最佳路徑所使用的規則，例如 RIP、OSPF、BGP。
- **Route（路由／路徑）**：從來源到目的地可採用的一條傳輸路徑，通常包含目的網路、下一站與度量值等資訊。
- **Routing Table（路由表）**：路由器儲存路徑資訊的表格，轉送封包時會查詢此表決定下一站。

### 3. 自訂一個網路距離運算法則，並說明原則與理由

可設計一個「成本值越小越好」的綜合路徑距離公式：

```text
路徑成本 = 0.4 × 延遲分數 + 0.3 × 頻寬分數 + 0.2 × 負載分數 + 0.1 × 跳數分數
```

原則如下：

- **延遲權重最高**：即時通訊、影音或互動式服務對延遲較敏感。
- **頻寬權重次高**：頻寬越大，傳輸大量資料時越有利，因此頻寬不足應提高成本。
- **負載納入考量**：避免所有流量都集中到同一條忙碌路徑。
- **跳數作為輔助**：跳數少通常代表路徑較短，但不一定代表速度最快，因此權重不宜過高。

此法則能比只看 Hop Count 更符合實際網路傳輸狀況，因為它同時考慮速度、壅塞與路徑長度。

### 4. 何謂備援路徑？應採用何種尋徑方式？為什麼？

**備援路徑**是指除了主要路徑之外，另外準備一條或多條可替代的路徑。當主要路徑發生故障、壅塞或中斷時，封包可以改走備援路徑，以維持網路連線不中斷。

此種架構較適合採用**動態尋徑**，尤其是 OSPF、EIGRP 等收斂速度較快的協定。原因是動態尋徑能自動偵測網路拓樸變化並重新計算路徑；若使用靜態尋徑，路徑中斷時通常需要人工修改設定，恢復速度較慢。

### 5. 比較靜態尋徑與動態尋徑的優缺點及差異

| 項目 | 靜態尋徑 | 動態尋徑 |
|---|---|---|
| 設定方式 | 由管理者手動設定 | 路由器透過協定自動學習 |
| 更新方式 | 不會自動更新 | 會依網路變化自動更新 |
| 優點 | 簡單、穩定、安全性較高、較不耗資源 | 適合大型網路、可自動備援、維護彈性高 |
| 缺點 | 網路變更時需人工修改，不適合大型複雜網路 | 較耗 CPU、記憶體與頻寬，設定較複雜 |
| 適用情境 | 小型網路、固定路徑、預設路由 | 中大型網路、多路徑、拓樸常變動環境 |

### 6. 距離向量法（Distance Vector）的運作機制與缺失

距離向量法的基本概念是：每台路由器只知道自己到各目的網路的距離，以及應該把封包交給哪一個鄰近路由器。路由器會定期把自己的路由表傳送給相鄰路由器，鄰近路由器再根據收到的資訊更新自己的路由表。

其運作特性包括：

- 以「距離」作為度量值，例如 RIP 使用 Hop Count。
- 只與鄰近路由器交換資訊。
- 常以週期性方式傳送路由表。
- 依據鄰居提供的資訊逐步推算到目的地的最佳路徑。

主要缺失如下：

- **收斂速度慢**：網路故障後，各路由器需要一段時間才會更新到一致狀態。
- **容易產生路由迴圈**：不同路由器可能彼此誤認對方仍有可用路徑。
- **可能出現無窮計數問題**：錯誤路由的距離值可能逐步增加，直到達到最大值。
- **定期更新耗費頻寬**：即使網路沒有變動，也可能持續交換路由表。
- **缺乏完整拓樸資訊**：每台路由器只知道鄰居告訴自己的資訊，較難掌握整體網路狀況。

### 7. 連線狀態（Link State）的尋徑機制，並比較其與距離向量的差異

連線狀態尋徑的基本方式是：每台路由器先偵測自己與鄰近路由器之間的連線狀態，然後透過 **LSA（Link-State Advertisement）** 將這些狀態資訊散佈給同一區域中的其他路由器。每台路由器取得完整拓樸資料後，會建立連線狀態資料庫，並使用 **SPF（Shortest Path First）** 演算法計算最短路徑樹，再產生自己的路由表。

| 項目 | 距離向量 | 連線狀態 |
|---|---|---|
| 資訊來源 | 鄰居提供的路由表 | 各路由器宣告的連線狀態 |
| 網路視野 | 較局部 | 較完整 |
| 更新方式 | 常見為週期性更新 | 多在狀態變更時觸發更新 |
| 演算法 | Bellman-Ford 類型 | Dijkstra SPF 類型 |
| 收斂速度 | 較慢 | 較快 |
| 資源需求 | 較低 | 較高 |
| 代表協定 | RIP | OSPF、IS-IS |

### 8. 混合式尋徑方式如何運作？與前兩種方式比較有何優點？

混合式尋徑結合距離向量與連線狀態的優點。它不像傳統距離向量只依賴週期性完整路由表交換，也不像完整連線狀態協定需要每台路由器都維護大量拓樸資訊；而是會與鄰居建立關係、維護可行路徑，並在路由變更時傳送較精簡的更新資訊。

其優點包括：

- **收斂速度較快**：可較快找到替代路徑。
- **較能避免路由迴圈**：可透過演算法判斷可行後繼路徑。
- **更新流量較少**：通常只在變更時傳送必要資訊。
- **兼具彈性與效率**：比傳統距離向量穩定，也比完整連線狀態方式節省部分資源。

代表例子為 **EIGRP**。

### 9. 尋徑過程中可能遭遇的問題及解決方案

尋徑過程常見問題如下：

1. **路由迴圈**  
   多台路由器因資訊不同步，可能互相把封包轉送給對方，造成封包在網路中循環。  
   **解決方式**：分離界線、路由毒化、觸發更新、保持計時。

2. **無窮計數問題**  
   距離向量協定中，錯誤路徑的距離值可能逐步增加，導致收斂變慢。  
   **解決方式**：定義最大度量值，例如 RIP 以 16 表示不可達。

3. **收斂時間過長**  
   網路拓樸改變後，各路由器需要一段時間才能取得一致資訊。  
   **解決方式**：使用觸發更新、選用收斂較快的協定，如 OSPF 或 EIGRP。

4. **路由資訊過期**  
   若路由器長時間未收到更新，可能仍保留錯誤路由。  
   **解決方式**：使用無效計時、保持計時與刪除計時等機制。

5. **更新流量過大**  
   大型網路若頻繁交換完整路由表，會造成額外頻寬負擔。  
   **解決方式**：路由彙總、區域化設計、觸發式或增量式更新。

6. **網路規模過大**  
   路由表過大會增加查表與維護成本。  
   **解決方式**：採用階層式設計，例如 OSPF Area、BGP 路由彙總與 AS 分層管理。

### 10. 何謂自律系統？內部尋徑與外部尋徑的差別

**自律系統（Autonomous System, AS）** 是由同一個管理單位控制、採用共同路由政策的一群網路與路由器。每個 AS 可具有唯一的 **AS Number**，用於網際網路上的跨 AS 識別。

| 項目 | 內部尋徑 | 外部尋徑 |
|---|---|---|
| 運作範圍 | 同一個 AS 內部 | 不同 AS 之間 |
| 主要目標 | 找出內部網路的最佳路徑 | 依政策與可達性選擇跨 AS 路徑 |
| 常見協定 | RIP、OSPF、IGRP、EIGRP、IS-IS | EGP、BGP |
| 度量重點 | 跳數、頻寬、延遲、成本等 | AS 路徑、政策、屬性、可達性 |

### 11. OSPF 運用哪種觀念降低尋徑交通量？其機制為何？

OSPF 運用**區域（Area）**的觀念降低尋徑交通量。

其機制是將大型自律系統切分成多個區域，每個區域內的路由器主要交換本區域的連線狀態資訊，而不必讓所有細節散佈到整個 AS。各區域通常透過骨幹區域 **Area 0** 互相連接，區域邊界路由器（ABR）可負責在區域之間傳遞與彙總路由資訊。

這樣可以減少 LSA 擴散範圍、降低路由表與連線狀態資料庫的規模，並減少每台路由器執行 SPF 計算的負擔。

### 12. EGP 協定以何種方式進行尋徑？請說明其運作機制

EGP 是早期用於自律系統之間的外部尋徑協定。它主要交換的是**網路可達性資訊**，而不是像 OSPF 那樣計算完整拓樸中的最短路徑。

EGP 的運作可概括為：

1. 外部閘道器先建立鄰居關係。
2. 透過詢問與回覆確認鄰居是否仍可到達。
3. 交換各自可到達的網路資訊。
4. 依據可達性資訊決定封包應該送往哪個外部閘道。

EGP 的限制是它較依賴早期網際網路的核心架構觀念，主要描述可達性，缺乏現代 BGP 所具備的完整政策控制與路徑屬性機制。

### 13. BGP 與 EGP 有何不同？請說明 BGP 的特色

BGP 是取代早期 EGP 的主要外部尋徑協定，常用於不同自律系統之間的路由交換。

BGP 的特色如下：

- **使用 TCP 傳輸**：BGP 使用 TCP 連線，可靠性較高，常用連接埠為 179。
- **路徑向量協定**：BGP 會利用 AS Path 等屬性描述路由經過哪些 AS。
- **避免路由迴圈**：若路由更新中已包含自己的 AS Number，即可判斷可能形成迴圈並拒絕該路由。
- **支援政策式路由**：可依 AS Path、Local Preference、MED 等屬性選擇路徑，而不只是看距離。
- **初始交換完整路由，之後採增量更新**：建立鄰居時交換完整路由表，後續主要傳送變更部分，降低更新流量。
- **不依賴核心網路架構**：BGP 不像早期 EGP 依賴核心閘道概念，較適合現代網際網路的分散式 AS 架構。
- **支援路由彙總與 CIDR**：有助於減少全球路由表規模。

因此，BGP 比 EGP 更適合大型、分散、需要政策控制的網際網路環境。
