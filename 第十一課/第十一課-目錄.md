# 第十一課：Router、Routing Table 與路由協定

這一課整理 `錄製 (17).m4a` 到 `錄製 (22).m4a` 的課堂重點，主軸是路由器如何在不同網路之間轉送封包、如何依照 `routing table` 選擇下一跳，以及靜態路由、動態路由、距離向量、連線狀態、混合式路由、AS/BGP 與常用路由診斷指令的基本觀念。

## 學習順序

1. 先讀 [[Router 與 Routing]]，建立 Router 為什麼能把不同網路連起來的基本觀念。
2. 再讀 [[Routing Table 路由表]]，理解 Router 查表後決定封包要從哪個 interface 送出。
3. 接著看 [[Static 與 Dynamic Routing]]，分清楚直接連接、靜態路由與動態路由三種來源。
4. 再讀 [[Distance Vector 路由]]，理解週期性交換路由資訊、以距離或 hop count 建表的做法。
5. 接著讀 [[Link State 與 OSPF]]，理解 flooding、Dijkstra 最短路徑與 `OSPF` 區域化的概念。
6. 再看 [[Hybrid Routing 與 EIGRP]]，把距離向量與連線狀態的優點如何合併起來。
7. 接著讀 [[Autonomous System 與 BGP]]，理解大型網際網路為什麼要分成 AS，以及 IGP/EGP 的層級差異。
8. 再讀 [[路由選擇 AD 與 Cost]]，理解 Router 面對多條路時會先比 AD，再比 cost。
9. 最後看 [[路由診斷指令]]，把 `ping`、`tracert`、`arp`、`route`、`pathping` 的用途串起來。

## 錄製 20 到 22 補強重點

- `錄製 (20)` 補強在 [[Router 與 Routing]]、[[Distance Vector 路由]]、[[Link State 與 OSPF]]、[[Hybrid Routing 與 EIGRP]]：包含 metric 不只看實體距離、三類動態路由比較、Dijkstra 水流比喻、負權重不適合 Dijkstra、Bellman-Ford 與 Distance Vector 的直覺。
- `錄製 (21)` 補強在 [[Autonomous System 與 BGP]]、[[Static 與 Dynamic Routing]]、[[路由選擇 AD 與 Cost]]：包含 AS/IGP/EGP/BGP、OSPF 自動交換路由表、`C` 與 `O` 路由表代號、cost 計算、容錯、負載平衡，以及 static route 與 dynamic route 混用。
- `錄製 (22)` 補強在 [[路由診斷指令]]：包含 `ping`、`tracert`、`arp -a`、`route print`、`pathping` 的用途與判讀順序。

## 錄音 17 新增重點

- 先用家用 Wi-Fi 與 Router 的影片建立直覺：連上 Wi-Fi 只代表連到家裡 Router，不一定代表 Router 外面到 Internet 的路徑正常。
- 補充 Router 的內外部角度：內部 `LAN` 常見 `192.168.x.x` private IP，外部 `WAN` 或 ISP 端則是 Internet 上看到的 public IP。
- 補充 Router 連接不同第三層網路，會依照目的地 `IP address` 與路由資訊，把封包送往下一台 Router 或下一個網路。
- 新增 `routing metric` 的觀念，路徑好壞不只看實際距離，也可能看 hop count、速度、穩定度、成本、壅塞程度等因素。
- 補充 `routing` 可以分成 `static routing` 與 `dynamic routing`。
- 新增 `Link State` 的基本概念：透過 flooding 分享連線狀態，再用 Dijkstra 最短路徑演算法計算較合適的路徑。
- 補充 `OSPF` 會用區域與 backbone area 降低大型網路的複雜度。

## 錄音 18 新增重點

- 整理三類動態路由做法：`Distance Vector`、`Link State`、`Hybrid`。
- 補充 `Distance Vector` 的做法：Router 週期性與相鄰 Router 交換路由表或距離資訊。
- 以 A、B、C、D 節點例子說明距離會一層一層被學到，路由表也會跟著更新。
- 補充 Distance Vector 的優點是簡單、資源需求低，小型網路容易使用。
- 補充 Distance Vector 的限制是大型網路更新成本高、資訊較不完整，收斂可能比較慢。

## 錄音 19 新增重點

- 以 `Routing Table` 影片說明：路由表像地圖或導航，記錄封包到目的地時應走哪條路。
- 補充路由表三種來源：`directly connected route`、`static route`、`dynamic route`。
- 說明 Router interface 設定 `IP address` 與 `subnet mask` 後，會自動產生直接連接路由。
- 補充靜態路由需要網管人員手動設定目的網路與下一跳，例如透過 `125.0.0.2` 去到 `174.16.0.0`。
- 補充靜態路由通常要雙向設定，否則可能只有單方向能送達，回程找不到路。
- 補充動態路由由相鄰 Router 透過協定交換資訊，常見協定包含 `RIP`、`OSPF`、`BGP`、`IS-IS`、`EIGRP`。

## 錄音 20 新增重點

- 補充 routing metric 不只是實體距離，也可能納入 hop count、delay、bandwidth、load、reliability、cost 等因素。
- 重新整理 `Link State` 與 `Distance Vector` 的差異：前者像每台 Router 都有完整地圖後自己算路，後者像向鄰居詢問到目的地的距離。
- 補充 Dijkstra 最短路徑演算法的直覺：從起點開始，每次挑目前距離最小的未確認節點，再更新鄰居距離。
- 補充 Dijkstra 適合邊權重為非負數的情境；若有負權重，應改用 Bellman-Ford 類型演算法。
- 補充 Distance Vector 背後的 Bellman-Ford 概念：到目的地的成本等於「到某鄰居的成本」加上「鄰居到目的地的成本」，再選最小者。
- 說明 Distance Vector 可能出現 routing loop、count to infinity、收斂慢等問題。
- 補充避免迴圈的做法：限制最大 hop count、split horizon、poison reverse、hold-down timer、triggered update。

## 錄音 21 新增重點

- 補充大型網路不可能讓每台 Router 都記住全世界所有細節，因此會用 `Autonomous System` 將網路分成可管理的區塊。
- 說明 `IGP` 用在同一個 AS 內部，例如 `RIP`、`OSPF`、`EIGRP`；`EGP` 用在不同 AS 之間，現代代表是 `BGP`。
- 補充 `RIP` 使用 hop count，常見最大 hop count 為 15，16 可視為不可達。
- 補充 `OSPF` 是 Link State 協定，具備 area 概念，適合大型網路。
- 補充 `BGP` 會記錄經過的 AS path，可用來避免迴圈，也能依政策選路。
- 以台北、台中、高雄 Router 範例說明靜態路由、動態路由、OSPF 自動交換路由表、容錯、負載平衡。
- 補充 `Administrative Distance` 與 `cost`：若同一目的地有不同來源的路由，先比 AD；AD 相同時再比 cost。
- 說明可混用靜態與動態路由，例如為了安全讓特定目的主機走指定路徑，其他流量仍交給 OSPF。

## 錄音 22 新增重點

- 補充 `ping` 可測試目的主機是否有回應，但有些設備會因安全或防火牆設定而不回應。
- 補充 `tracert` 會從 TTL=1 開始逐步增加，顯示封包到目的地中間經過哪些節點。
- 補充 `arp -a` 可查看本機 ARP cache，了解 IP address 與 MAC address 的對應。
- 補充 `route print` 可查看本機 routing table，必要時也可新增或刪除路由。
- 補充 `pathping` 類似強化版 `tracert`，會用更多測試次數統計路徑上的封包遺失比例。
