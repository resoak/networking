# Static 與 Dynamic Routing

## 為什麼要分 Static 與 Dynamic

Router 必須知道封包要怎麼走，但路由資訊不一定都從同一個地方來。

課堂整理成三種：

1. 直接連接路由。
2. 靜態路由。
3. 動態路由。

其中靜態與動態的差別，是路由資訊到底由人設定，還是由 Router 透過協定自動交換。

## Static Routing

`Static routing` 是由網管人員手動設定路由。

例如要讓 Router 知道：

> 如果要去 `174.16.0.0`，下一跳請送到 `125.0.0.2`。

這種路由寫進去後，Router 就會照著做。

### 優點

- 概念簡單。
- 可控性高。
- 小型或固定拓撲的網路很方便。
- 不需要 Router 之間持續交換大量路由資訊。

### 缺點

- 網路變動時不會自動修正。
- 網路很大時，人工維護成本很高。
- 容易漏掉回程路由。

## 靜態路由要注意方向

錄音 19 特別提到：路由通常要考慮雙向。

假設 `192.168.0.0` 要連到 `174.16.0.0`，你在第一台 Router 設定了往 `174.16.0.0` 的 static route，這只代表去程知道怎麼走。

但 `174.16.0.0` 那邊如果不知道怎麼回到 `192.168.0.0`，回程封包仍然可能失敗。

所以雙向溝通常常需要兩邊都有可用路由。

## Dynamic Routing

`Dynamic routing` 是由 Router 透過 routing protocol 自動交換路由資訊。

課堂中提到的常見協定包括：

- `RIP`
- `OSPF`
- `BGP`
- `IS-IS`
- `EIGRP`

動態路由的核心是：

> Router 彼此告訴對方自己知道哪些網路、路徑狀態如何，再把資訊更新進 routing table。

錄音 21 用台北、台中、高雄三台 Router 的例子說明：網段越多、Router 越多，手動維護 static route 就越容易漏掉或設錯。啟動 `OSPF` 這類動態路由後，Router 可以自動交換自己知道的網段，讓 routing table 自動補齊。

### 優點

- 適合大型網路。
- 網路狀態變動時，可以自動更新路由。
- 可以根據協定的 metric 選出較合適的路徑。

### 缺點

- 需要額外協定與設定。
- Router 需要花資源交換與計算路由資訊。
- 協定本身比 static route 複雜。

## 比較表

| 項目 | Static Routing | Dynamic Routing |
|---|---|---|
| 路由來源 | 人工設定 | Router 之間交換資訊 |
| 網路變動 | 不會自動更新 | 可依協定自動更新 |
| 適合環境 | 小型、固定拓撲 | 大型、常變動拓撲 |
| 管理成本 | 網路越大越高 | 初期設定較複雜，但可自動維護 |
| 常見例子 | 手動指定下一跳 | `RIP`、`OSPF`、`EIGRP` |

## 靜態與動態可以混用

實務上不一定只能二選一。

課堂例子提到，一般路由可以交給 `OSPF` 自動維護；但如果某台重要主機或某段網路有安全考量，也可以手動加 static route，指定它走比較安全的路徑。

這時 Router 會依路由來源的優先順序與 metric 選路。常見判斷方式是：

1. 先看 `Administrative Distance`，數字越小越優先。
2. 如果 AD 相同，再看協定內部的 `cost`。
3. 如果 cost 也相同，可能做負載平衡。

例如 `OSPF` 常見 AD 是 `110`，如果手動設定一條 static route 的 AD 為 `50`，Router 會優先採用 AD 較小的 static route。

## AD 與 Cost 的選路順序

錄音 21 補充了更詳細的選路邏輯：

1. **先比 Administrative Distance (AD)**：數字越小越優先
2. **如果 AD 相同，再比 cost**：數字越小越優先
3. **如果 cost 也相同，可能做負載平衡**：輪流使用多條路徑

課堂例子中，`OSPF` 的 AD 是 `110`。如果手動設定一條 static route，並把 AD 設成 `50`，Router 會先採用 AD 較小的 static route；只有指定路徑不可用時，才回到其他可用路徑。

這裡不需要背所有協定的 AD 數字，先抓住核心觀念：**AD 是比較路由來源，cost 是比較同一協定內的路徑成本。**

## 容錯設計

錄音 21 強調了容錯的重要性：

> 多拉幾條線不只是浪費錢，也可能是為了：
> - `fault tolerance`：某條專線斷掉時，還有其他路徑可走。
> - `load balancing`：兩條成本相同的路徑可以分散流量。

以台北、台中、高雄三個辦公室為例，多拉一條專線雖然成本較高，但某條線斷掉時，仍可改走另一條路；如果兩條路 cost 相同，也可以把流量分散到不同路徑。

## 實務應用

錄音 21 提供了實務應用場景：

> 假設我們有台北、台中、高雄三個辦公室之間都拉專線時，台北到高雄可以：
> - 直接走台北到高雄。
> - 先走台北到台中，再從台中到高雄。

如果網路有很多網段，全部用 static route 容易漏設或設錯；使用 `OSPF` 這類 dynamic routing 可以讓 Router 自動交換路由表。但對於特定重要主機，也可以額外加 static route，指定它走比較安全的路徑。

## 怎麼記

- Static：人寫的路。
- Dynamic：Router 自己學到的路。
- AD：比較不同路由來源，越小越優先。
- Cost：比較同一協定內的路徑成本，越小越優先。
- 多路徑可用來容錯，也可用來負載平衡。
- 靜態路由簡單但不會自動修正；動態路由複雜但能適應變化。
- 大型網路常用 Dynamic 為主，Static 補特殊需求。
