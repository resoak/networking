# 快速記憶卡

## Router 與 Routing

- `Router`：連接不同網路，替封包決定下一站。
- `Routing`：封包從來源到目的地之間的選路過程。
- 封包通常經過一台一台 Router，不是一次直接到目的地。
- Wi-Fi 已連線只代表連到本地 Router，不代表 Router 外面到 Internet 一定正常。

## LAN/WAN 與內外部 IP

- `LAN`：家裡內部網路，手機、筆電、桌機連到 Router 的這一側。
- `WAN`：Router 對外連到 ISP / Internet 的這一側。
- `192.168.x.x` 常見於 private IP，只在內部網路有意義。
- Public IP 是外面 Internet 主機看到、用來找到你家 Router 外部的位置。
- Wi-Fi 連上但不能上網，可能是內部 LAN 正常、外部 WAN 或 ISP 出問題。

## Routing Metric

- Metric 是衡量路徑好壞的分數或成本。
- 不一定只看物理距離。
- 可能看 `hop count`、速度、穩定度、成本、壅塞、錯誤率。
- 最佳路徑是依協定定義出的「較適合」路徑。

## Routing Table

- `routing table` 像 Router 的地圖或導航。
- Router 查目的地 `IP address` 後，決定出口 interface 或下一跳。
- 找不到可用路由時，封包可能被丟棄。
- 路由表來源：directly connected、static、dynamic。

## Directly Connected Route

- Router interface 直接連到的網路。
- 設定 interface 的 `IP address` 與 `subnet mask` 後自動產生。
- 只知道直接相連的網路，不代表知道更遠的網路。

## Static Routing

- Static route 是人工設定的路。
- 優點：簡單、可控。
- 缺點：網路變動時不會自動修正。
- 注意：去程有路，不代表回程也有路；雙向溝通常要兩邊都有路由。

## Dynamic Routing

- Dynamic route 是 Router 透過 routing protocol 自動學到的路。
- 適合大型或常變動網路。
- Router 會交換路由資訊並更新 routing table。
- 常見協定：`RIP`、`OSPF`、`BGP`、`IS-IS`、`EIGRP`。

## Distance Vector

- 跟鄰居交換「到哪裡多遠」。
- 常見 metric：`hop count`。
- 優點：簡單、資源需求低。
- 缺點：大型網路更新成本高、收斂可能慢。
- 代表：`RIP`。

## Link State 與 OSPF

- 分享「我連到誰、連線成本如何」。
- 透過 flooding 傳播連線狀態。
- 用 Dijkstra 類型演算法算較佳路徑。
- `OSPF` 是常見 Link State 協定。
- `OSPF` 會用 area 與 backbone area 降低大型網路複雜度。

## Hybrid 與 EIGRP

- Hybrid：混合式路由做法。
- 結合 Distance Vector 與 Link State 的部分優點。
- `EIGRP` 是代表例子。
- 這一課先記分類，不急著背完整細節。

## AS、IGP、EGP、BGP

- `AS`：Autonomous System，可獨立管理的大型路由區塊。
- `IGP`：同一個 AS 內部使用的路由協定。
- `EGP`：不同 AS 之間使用的路由協定。
- `BGP`：現代 Internet 上 AS 之間的主要路由協定。
- `BGP` 會看 AS path，可協助避免路由迴圈，也能依政策選路。

## AD 與 Cost

- `Administrative Distance`：比較不同路由來源的優先權，越小越優先。
- `Cost`：同一協定內比較路徑成本，越小越優先。
- 選路常見順序：先比 AD，再比 cost。
- Cost 相同時，可能做負載平衡。
- 多條路徑可提供容錯，某條線斷掉時改走備援路。

## Routing Loop

- Routing loop：路由資訊錯誤或未收斂時，封包可能在 Router 間繞圈。
- Count to infinity：距離數字一直增加到最大值才判定不可達。
- 常見改善：最大 hop count、split horizon、poison reverse、hold-down timer、triggered update。

## 診斷指令

- `ping`：測試目的主機是否有回應。
- `tracert`：追蹤封包經過哪些 Router。
- `arp -a`：查看 IP 與 MAC address 的對應。
- `route print`：查看本機 routing table。
- `pathping`：追蹤路徑並統計丟包情況。
