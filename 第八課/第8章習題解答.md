### 選擇題
1. `B`：目前主流仍是 IPv4。
2. `D`：下一代為 IPv6。
3. `C`：IPv4 是 32 位元。
4. `D`：題目選項中最接近 A 類的是 1~127；嚴格教材常寫 1~126，127 保留做 loopback。
5. `B`：190.x.y.z 為 B 類。
6. `C`：127.0.0.1 用來測試本機 TCP/IP。
7. `D`：127.0.0.1 不屬於私有位址。
8. `C`：50 個子網至少需借 6 位元。
9. `A`：若依 CIDR 最小共同涵蓋範圍，可視為 132.86.0.0 起的較大區塊。
10. `C`：四個連續 /24 可整成 `/22`，遮罩 255.255.252.0。
11. `C`：剩 5 個主機位元，可用主機數 30。
12. `A`：192.0.0.0 是合法遮罩。
13. `D`：DHCP。
14. `C`：完整網域名稱。
15. `A`：`com.jp` 不合日本常見命名規則，通常為 `co.jp`、`ac.jp` 等。
16. `C`：IHL 以 32 位元為單位。
17. `C`：IP 分片 offset 以 8 位元組為單位。
18. `D`：教材常寫 TTL（Time To Live），名稱上與時間有關；但實際轉送時通常由路由器逐跳遞減。
19. `C`：4 筆記錄需 3 + 16 = 19。
20. `B`：時間戳記多以毫秒記錄。
21. `A`：嚴格源路徑會規定每一跳。
22. `D`：IPv6 是 128 位元。
23. `C`：任點傳播（Anycast）不是無規則傳送，而是依路由機制送往最近或最適合的節點。
24. `C`：不可出現兩次 `::`。
25. `B`：此題較安全的寫法是：IPv6 的即時服務與優先權處理不能只單看單一欄位，Traffic Class、DSCP 與其他機制需搭配理解。

### 問答題
#### 1. 名詞英文全名
- IP = **Internet Protocol**
- ICANN = **Internet Corporation for Assigned Names and Numbers**
- NAT = **Network Address Translation**
- IETF = **Internet Engineering Task Force**
- CIDR = **Classless Inter-Domain Routing**
- DHCP = **Dynamic Host Configuration Protocol**
- DNS = **Domain Name System**
- FQDN = **Fully Qualified Domain Name**
- TLD = **Top Level Domain**
- TWNIC = **Taiwan Network Information Center**
- IHL = **Internet Header Length**
- TOS = **Type of Service**
- DS = **Differentiated Services**
- DSCP = **Differentiated Services Code Point**
- QoS = **Quality of Service**
- ECN = **Explicit Congestion Notification**
- MTU = **Maximum Transmission Unit**
- TTL = **Time To Live**
- ng = **next generation**
- IoT = **Internet of Things**
- IID = **Interface Identifier**
- FP = **Format Prefix**

#### 2. 何謂 IP？IPv4 / IPv5 / IPv6 / IPng
- IP 是網際網路上的邏輯定址與封包遞送協定。
- IPv4：32 位元；IPv5：非主流實驗版本；IPv6 / IPng：128 位元，為解決位址不足而生。

#### 3. Octet 與 Byte 差異
- Octet 明確指 8 位元；Byte 在現代幾乎也視為 8 位元。

#### 4. IPv4 位址分類方式
- 可依 A/B/C/D/E 類分類，也可依公有/私有、單播/廣播/群播、CIDR 分類。

#### 5. 何謂網域位址？
- 用來識別某一網段本身；主機位元全為 0 時即為網路位址。

#### 6. 何謂廣播位址？
- 用來對某一網段內所有主機廣播；主機位元全為 1 即為廣播位址。

#### 7. 何謂 Multicasting？何謂 MRouter？
- Multicasting 是送給特定一群接收者；MRouter 是支援多播路由的路由器；IPv4 多播使用 Class D 位址。

#### 8. 何謂網路遮罩？
- 用來區分 IP 中哪些位元屬於網路、哪些屬於主機；與 IP 做 AND 可得網路位址。

#### 9. IPv4 重要保留位址
- `0.0.0.0`、`127.0.0.1`、`255.255.255.255`，以及私有位址區塊 10/8、172.16/12、192.168/16。

#### 10. 何謂 NAT？
- 把內部私有 IP 轉換成外部公有 IP，可節省公有位址並隔離內外部網路。

#### 11. 合法 IP 與類型判斷
- `160.32.88.250`：合法，B 類。
- `224.2.25.38`：合法，D 類（多播）。
- `0.0.0.0`：保留位址。
- `500.2.87.62`：不合法。
- `192.168.132.58`：合法，C 類私有位址。
- `210.32.96.170`：合法，C 類公有位址。
- `140.134.256.30`：不合法。
- `10.25.125.5`：合法，A 類私有位址。
- `127.0.0.1`：合法，Loopback。
- `921.921.921.921`：不合法。

#### 12. 子網路規劃方式與目的
- 從主機位元借位元切出更多子網。
- 目的：分散管理、降低廣播、提升安全與效率。

#### 13. 【依實際環境】子網路規劃作答範例
- 先統計部門/樓層/設備數，再為每個區段分配對應大小的子網，並預留成長空間。

#### 14. 何謂 CIDR？
- CIDR 是無類別網域間路由，以 `/前綴長度` 表示網路大小，提升位址使用效率。

#### 15. 何謂 Supernet？
- 把多個連續網段整併成較大的區塊，可減少路由表項目；與 Subnet 相反。

#### 16. CIDR 區塊選取規範
- 位址數量通常需為 2 的次方，起始位址必須對齊該區塊大小。

#### 18. 何謂 DHCP？
- DHCP 會自動發放 IP、遮罩、閘道器、DNS 等設定，適合大量終端設備環境。

#### 19. 何謂 DNS？
- DNS 把人可讀的網域名稱轉成 IP 位址。

#### 20. 網域名稱、根網域、頂層網域、FQDN
- Root Domain 是最頂端；TLD 如 `.com`、`.tw`；FQDN 如 `www.example.com.`。

#### 21. 何謂短網址？何謂泛用型網域名稱？
- 短網址是把長網址映射成較短網址；gTLD 如 `.com`、`.org`、`.net`。

#### 22. Domain 與 Zone 差異
- Domain 是命名空間概念；Zone 是 DNS 伺服器實際管理的一段資料範圍。

#### 23. DNS 查詢有哪些類型？
- 遞迴查詢：被問的一方要幫你問到底。
- 反覆/迭代查詢：被問的一方只告訴你下一個該問誰。

#### 24. 何謂動態 DNS？
- IP 變動時自動更新 DNS 對應，適合家用寬頻與遠端監控。

#### 25. IP 標頭欄位定義
- 依圖中的 Wireshark 畫面，此 IP 標頭可整理為：
  - **Version = 4**：表示此封包使用 IPv4。
  - **IHL / Header Length = 5**：代表 IP 標頭長度為 `5 × 32 bits = 20 bytes`。
  - **Service Type = Routine / Delay-Normal / Throughput-Normal / Reliability-Normal**：表示此封包未特別要求高優先權、低延遲或高可靠度，屬一般傳輸。
  - **Total Length = 54 octets**：整個 IP 封包總長度為 54 Bytes，包含 IP 標頭與資料。
  - **Fragment ID = 9276**：分片識別碼；若封包被切成多片，接收端可用此欄位判斷哪些分片屬於同一原始封包。
  - **Flags = Do not fragment, Last fragment**：表示此封包設定為不要分片，且目前也是最後一片；因為沒有實際分片，所以可視為單一完整封包。
  - **Fragment Offset = 0**：表示這個封包不是某個分片封包中的中間段或尾段，而是從開頭開始。
  - **TTL = 255**：封包最多可經過 255 次路由轉送，每經過一台路由器會遞減，降到 0 就會被丟棄。
  - **Protocol = TCP (0x06)**：表示 IP 資料負載上層使用的是 TCP 協定。
  - **Header Checksum = 0xF5D2**：用來檢查 IP 標頭在傳送過程中是否發生錯誤。
  - **Source IP = 140.134.36.56**：來源主機位址。
  - **Destination IP = 140.134.36.110**：目的主機位址。
  - **Options = None**：此封包沒有使用 IP 選擇項，因此標頭維持最基本的 20 Bytes。
- 若要補圖中的空格，可直接寫成：
  - `Version = 4`
  - `Header Length = 5 (20 bytes)`
  - `Service Type = Normal`
  - `Total Length = 54 bytes`
  - `Fragment ID = 9276`
  - `Flags = Do not fragment`
  - `Fragment Offset = 0`
  - `TTL = 255`
  - `Protocol = TCP (0x06)`
  - `Checksum = 0xF5D2`
  - `Source IP = 140.134.36.56`
  - `Destination IP = 140.134.36.110`
  - `Options = None`

#### 26. IP 封包優先權如何設定？
- IPv4 常透過 TOS / DSCP；IPv6 常透過 Traffic Class / DSCP 搭配其他機制處理，Flow Label 可用於識別特定 flow，但不宜視為唯一依據。

#### 27. IHL 與 TL 差異
- IHL：Header 長度，單位為 32 位元字。
- TL（Total Length）：整個 IP 封包長度，單位為位元組。

#### 28. 何謂 MTU？
- 單一鏈路可承載的最大封包大小；超過時需分片或調整。

#### 29. FO 與 ID 如何運作？
- FO 表示分片在原封包中的位置，單位是 8 Bytes；ID 則讓接收端知道哪些分片屬於同一原始封包。

#### 30. 1 補數演算法與標頭檢查碼
- 1 補數演算法的作法，是先把 **IP 標頭以 16 位元為一組** 依序相加；若加總結果超過 16 位元，就把最高位的進位加回最低位；全部加完後，再對結果取 **1 補數**，就是標頭檢查碼。
- 依第 25 題封包內容，可將 IP 標頭寫成下列 16 位元資料（檢查碼欄位先視為 `0000`）：
  - `4500`
  - `0036`
  - `243C`
  - `4000`
  - `FF06`
  - `0000`
  - `8C86`
  - `2438`
  - `8C86`
  - `246E`
- 逐步相加如下：
  - `4500 + 0036 = 4536`
  - `4536 + 243C = 6972`
  - `6972 + 4000 = A972`
  - `A972 + FF06 = 1A878 → A879`（超過 16 位元，將進位 1 加回）
  - `A879 + 0000 = A879`
  - `A879 + 8C86 = 134FF → 3500`
  - `3500 + 2438 = 5938`
  - `5938 + 8C86 = E5BE`
  - `E5BE + 246E = 10A2C → 0A2D`
- 最後對 `0A2D` 取 1 補數：
  - `FFFF - 0A2D = F5D2`
- 所以本題 **IP Header Checksum = `0xF5D2`**，與圖中 Wireshark 顯示結果一致。

#### 31. IP 選擇項 Code 欄位定義
- Code 通常由 `1 bit Copy Flag + 2 bits Class + 5 bits Number` 組成。

#### 32. Record Route 選項
- 功能：記錄封包經過的路由節點。

#### 33. Source Routing 選項
- 讓傳送端指定封包要經過的路徑，可分嚴格與鬆散兩種。

#### 34. Strict 與 Loose Source Routing 差異
- Strict：每一跳都被明確指定。
- Loose：只規定必經節點，中間可自行尋徑。
- `137 = 10001001₂`，`131 = 10000011₂`。

#### 35. Timestamp 選項目的與時間基準
- 用來記錄封包經過節點的時間資訊，常以毫秒表示，通常以 UTC / GMT 為基準。

#### 36. Timestamp 欄位與運作機制
- 常含 Type、Length、Pointer、Overflow/Flag、Timestamp Data；封包經過節點時依規則寫入時間資訊。

#### 37. IPv6 位址空間與表示方式
- 128 位元，位址空間極大；可分單播、多播、任點播；用 16 進位表示，可用 `::` 壓縮。

#### 38. 何謂 IoT？
- IoT 是把感測器、裝置、網路與雲端服務整合起來，例如智慧家電、智慧監控、智慧工廠。

#### 39. IPv6 如何透過 IPv4 網路通訊？
- 主要做法是 **Tunneling**：把 IPv6 封包封裝在 IPv4 封包中傳送。

#### 40. IPv4 相容位址與 6to4 位址
- IPv4 相容位址：早期遷移用途，現在多視為過時或極少使用。
- 6to4 位址：依 IPv4 位址自動產生部分 IPv6 位址，屬過渡技術，現今已較少使用。

#### 41. IPv6 與 IPv4 標頭異同
- IPv6 標頭較精簡、固定長度、沒有標頭檢查碼；IPv4 有可變長度選項。

#### 42. IPv4 選擇項與 IPv6 延伸標頭差異
- IPv4 Options 直接掛在基本標頭後，效率較差。
- IPv6 改用 Extension Header，需要時才附加，較模組化、效能較佳。
