# 小測驗

## 題目

1. `IPv4` 位址長度是幾 bits？
2. `DHCP` 的用途是什麼？
3. `127.0.0.1` 通常代表什麼？
4. `255` 的 8-bit 二進位是什麼？
5. `132` 可以拆成哪兩個 2 的次方權重？
6. `/30` 代表前幾位是網路位元？
7. 判斷兩台主機是否同網段，最常想到哪個運算？
8. `DNS` 的正向查詢是什麼意思？
9. `DNS` 查詢如果本地 DNS 沒有快取答案，通常會往哪些層級問？
10. `A Record` 和 `AAAA Record` 分別記錄什麼？
11. `CNAME Record` 的用途是什麼？
12. `DDNS` 適合解決什麼問題？
13. `TTL` 在 IPv4 標頭中的主要用途是什麼？
14. `Protocol` 欄位用來表示什麼？
15. IPv4 分片重組會用到哪三個欄位？
16. `Header Checksum` 檢查的是整個資料內容，還是 IPv4 標頭？

## 答案

1. **32 bits**
2. **自動分配 IP 位址**
3. **Loopback / 本機測試位址**
4. **11111111**
5. **128 和 4**
6. **前 30 位**
7. **AND 運算**
8. **名稱 → IP 位址**
9. **Root → TLD → Authoritative DNS Server，最後回到本地 DNS / 使用者**
10. **A 記錄 IPv4；AAAA 記錄 IPv6**
11. **把別名指向正式名稱**
12. **IP 會變時，仍希望用固定名稱找到設備或服務**
13. **防止封包在網路中無限繞路；每過 router 會減少，到 0 就丟掉**
14. **指出 IPv4 後面承載的是哪個上層協定，例如 ICMP、TCP、UDP**
15. **Identification、Flags、Fragment Offset**
16. **IPv4 標頭**
